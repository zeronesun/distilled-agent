- **架构根因**（Why-2）：L3·架构设计中"评估扩展性和失败模式"规律提炼过于宽泛，未区分场景——用户在做小型内部工具时不需要扩展性评估，在做生产系统时才需要
- **归属层**：L3
- **修正方案**：更新 L3·架构设计规律为：
  - 原：评估扩展性和失败模式，不只写 happy path
  - 新：生产系统 → 评估扩展性和失败模式；内部工具 → 评估可维护性和数据安全
- **下次验证**：下次架构设计任务时，前馈清单中的 P2 预判应区分系统类型，询问用户确认
