# L2/L3 初始化输出示例

本文件展示一次典型的 L2/L3 初始化对话。用户在发送提炼指令后，Agent 基于对用户的历史了解，输出结构化的 L2 和 L3 草案。

> **说明**：本示例中的用户偏好为虚构，仅用于展示输出格式和提炼粒度。你的 Agent 会根据你们的历史对话生成属于你的版本。

---

## Agent 的提炼输出

基于我对你的当前了解，以下是提炼结果：

### [L2·用户底层逻辑]

- **沟通偏好**：简洁直接，不要客套铺垫；边界清晰，说"不"即停
- **质量标准**：代码优先附带验证方法；输出须可复用，不满足一次性正确
- **决策模式**：风险前置，先暴露完整问题再谈方案；二阶评估优先于直觉
- **学习模式**：跨领域理论迁移；知行合一，理解必须落地为操作

### [L3·场景核心规律]

- **短篇创作**：
  - 情绪定调优先于情节铺展
  - 结尾留白，不做价值总结或道德升华
  - 白描建立画面感，不依赖比喻渲染

- **代码输出**：
  - 先写测试或验证方法，再写实现
  - 功能完整优先于注释详尽
  - 不确定的行为标注"待验证"，不硬编

- **架构设计**：
  - 先定义系统边界与数据流，再选技术栈
  - 评估扩展性和失败模式，不只写 happy path
  - 输出包含风险分级（P0-P3）及应对

- **问题排查**：
  - 先定位日志/运行现场根因，不凭经验猜测
  - 不确定处标注"待验证"
  - 给出可复用的排查步骤，不只给结论

---

## 用户可以做的审核动作

1. **逐条确认**：每条原则问自己——"这真的是我的稳定偏好吗？"
2. **修正偏差**：直接告诉 Agent 哪条不对，应该改成什么
3. **补充缺失**：如果你觉得某个维度/场景遗漏了关键原则，直接说
4. **删除不实**：如果是 Agent 硬编的（你没有这个偏好），直接删掉

---

## 常见修正示例

| Agent 的原始提炼   | 用户的修正                               | 说明                               |
| :----------------- | :--------------------------------------- | :--------------------------------- |
| 沟通偏好：简洁直接 | 沟通偏好：简洁但必须有充分上下文         | 用户发现自己需要上下文，不只是短   |
| 短篇：结尾留白     | 短篇：结尾留白；但系列故事可以有温和收束 | 用户补充了例外情况                 |
| 代码输出：测试优先 | 代码输出：快速原型时可以后补测试         | 用户区分了原型和正式输出的不同标准 |

---

## 写入指令

审核通过后，发送：
> 将以上确认后的 L2 和 L3 写入 memory。

Agent 会将它们存入记忆的主体区，按 [L2·用户底层逻辑] 和 [L3·场景核心规律] 分节点存储。
