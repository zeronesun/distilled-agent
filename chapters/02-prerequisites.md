

# 第二章：适用条件与准备工作

## 2.1 这套方法对 Agent 平台的最低要求

在动手之前，先确认你使用的 Agent 平台是否具备以下能力。这些是架构落地的物理基础。

### 2.1.1 必需能力（缺一则无法完整实施）

| 能力               | 说明                                   | 用途                                                     |
| :----------------- | :------------------------------------- | :------------------------------------------------------- |
| **持久化记忆**     | 跨会话存储、读取、更新的记忆系统       | 承载 L1 元认知框架、L2 用户逻辑、L3 场景规律、积分控制表 |
| **程序性模块**     | 可创建、调用、更新的技能/Skill/GPTs 等 | 承载执行层（交付闸门、前馈启动、闭环梳理等）             |
| **足够上下文窗口** | 建议 100k tokens 以上                  | 容纳会话内的前馈清单、避坑记录、任务执行过程             |

如果你的平台以上三项都具备，可以完整落地本教程的全部内容。

### 2.1.2 加分能力（有则更强，无则降阶运行）

| 能力         | 说明                                         | 缺失时的替代方案                                 |
| :----------- | :------------------------------------------- | :----------------------------------------------- |
| **会话检索** | 搜索历史会话中的内容和纠正记录               | 在每次纠正后手动将关键教训写入记忆，作为前馈素材 |
| **工具调用** | Agent 可主动调用外部工具（搜索、代码执行等） | 范围受限但核心架构不受影响                       |

### 2.1.3 如果平台没有 Skill 功能怎么办

一些平台（如基础版 ChatGPT、简单聊天界面）没有技能模块。此时可采用 **"记忆模拟法"**：

- 将 Skill 的内容以结构化记忆条目存放，如 `[Skill-交付闸门] 内容：...`
- 在 L1 元认知框架中加入强制规则："每次交付前，必须读取并执行标记为 [Skill-*] 的所有记忆条目"
- 缺点：占用记忆容量，且需 Agent 主动检索；优点：逻辑完全保留，适用于轻度使用场景。

## 2.2 自我评估：你适合使用本教程吗

### 2.2.1 强烈推荐的使用者画像

- **高精度协作用户**：你对 Agent 的输出有稳定质量要求（如写作风格、代码规范）
- **重复任务用户**：你频繁与 Agent 进行同类任务（如每日/每周相似工作流）
- **系统思维用户**：你愿意投入前期搭建成本，换取长期协作效率
- **边界明确用户**：你能清晰表达喜好和红线，Agent 犯错时你愿意指出

### 2.2.2 不太适合的使用者

- 只进行一次性、无复用需求的随机问答
- 使用场景跨越非常广泛、难以归纳出稳定规律
- 不希望与 Agent 建立长期协作关系

## 2.3 开始前的准备工作

在正式部署之前，完成以下四项准备，可显著降低后续调整成本。

### 2.3.1 盘点当前 Agent 状态

运行以下两条指令，记录当前状态作为基线：

```
列出你当前拥有的所有 skill，以及它们的内容摘要。
```

```
列出你 memory 中当前存储的所有内容，并评估使用率。
```

**记录要点**：
- 当前有多少个 skill？它们是否仍在使用？
- memory 使用率百分之几？内容是否已经混杂？
- 是否有大量过时信息？

### 2.3.2 列出你想固化的偏好与规则

用一个简单的表格，列出你希望 Agent 长期遵守的原则。这一步帮你从模糊的"感觉"中提炼出可操作的规则：

| 维度     | 你的原则               | 示例                   |
| :------- | :--------------------- | :--------------------- |
| 沟通风格 | 简洁直接，不用客套话   | 不输出"希望能够帮到您" |
| 输出格式 | 代码优先，附带验证方法 | 输出方案时包含测试步骤 |
| 决策偏好 | 风险前置，先暴露问题   | 先列风险再谈收益       |
| 审美取向 | 冷感叙事，拒绝 AI 总结 | 小说结尾留白，不升华   |

不需要一次穷尽，4-6 条核心原则即可。后续 Agent 会在闭环中帮你不断补充。

### 2.3.3 回想 3-5 个你频繁执行的任务类型

- 例如：写短篇小说 / 代码审查 / 运维排查 / 架构设计 / 数据分析
- 每个类型试写一句话，描述你认为这类任务"做对"的关键规律是什么

**示例**：
- 短篇创作："情绪定调优先于情节铺展，结尾必须留白。"
- 代码审查："先找架构问题，再看命名规范；每发现一类问题给修改建议。"
- 问题排查："先看日志定位根因，不等猜测；不确定时标注'待验证'。"

这些将直接成为你的 L3 场景规律初始内容。

### 2.3.4 备份当前状态

在开始部署前，**备份你当前的记忆和技能状态**。具体方法取决于你的平台：

- 如果平台支持导出，先导出一份
- 如果不支持，手动复制当前 memory 和 skill 列表到本地文档
- 如果有版本管理（如部分平台的记忆快照），创建一个恢复点

这一步的目的是：万一部署过程中出现混乱，你可以恢复到初始状态重新开始。

## 2.4 准备完成的标志

当你可以回答以下三个问题时，准备阶段完成，可以进入第三章：

1. 我的 Agent 平台是否具备持久化记忆 + 技能模块？（是 / 否 / 部分具备）
2. 我梳理出了几条核心偏好和原则？（建议 ≥4 条）
3. 我列举出了几个高频任务类型，并对每个类型有了初步的"规律判断"？（建议 ≥3 个）
