# Обучение AI-агента по инженерной кибернетике Цянь Сюэсэня

> Не учите AI новым знаниям. Используйте **предварительную связь, интегральное управление, иерархическое разделение и самоконтроль** — четыре концепции — для перестройки архитектуры памяти и навыков агента, превращая его из пассивного инструмента в самокорректирующийся и эволюционирующий движок сотрудничества.

---

## Что это

Полный, воспроизводимый практический учебник.

Он адаптирует ключевые идеи *Инженерной кибернетики* Цянь Сюэсэня для современных платформ AI-агентов (Hermes, Claude с MCP, пользовательские GPT и т. д.). Без добавления команд в prompt или введения новых моделей он использует **архитектурную реструктуризацию**, чтобы агент мог:

- 🎯 **Предварительная связь для предотвращения ошибок** — заранее загружает ваши предпочтения и исторические уроки перед началом задачи
- 🔧 **Интегральное управление для предотвращения колебаний** — разовая жалоба получает временное исправление; та же обратная связь дважды вызывает системное изменение
- 🧹 **Иерархическое разделение** — память хранит только «почему»; навыки хранят только «как»; никогда не смешиваются
- 🚦 **Шлюз доставки с самоконтролем** — пятикратный контроль качества выполняется перед любым выводом; некачественные результаты никогда не доставляются
- 🔄 **Эволюция с замкнутым циклом** — скажите «взлёт» раз в неделю, и агент автоматически проверяет и оптимизирует себя

---

## Быстрый старт

Отправьте эту команду своему агенту, и он автоматически завершит полное обучение (≈ 40 минут):

```
按 github.com/zeronesun/cybernetic-your-agent 的 scripts/deployment-commands.md 训练我。每步确认，完成后闭环测试。
```

**Требования**: Ваш агент должен иметь доступ к GitHub, возможность записи в память и создания навыков. Если это не выполнено, следуйте ручным шагам ниже.

1. Убедитесь, что ваша платформа агента имеет: **постоянная память** + **навыки / процедурные модули**
2. Прочитайте учебник (рекомендуется по порядку, или хотя бы прочитайте главу 1, чтобы понять подход)
3. Перейдите к [главе 5](chapters/05-deployment.md), скопируйте инструкции и развертывайте по шагам (≈ 1 час)
4. [Глава 6](chapters/06-daily-usage.md) расскажет, как использовать ежедневно и проверить работу

## Поддерживаемые платформы

- ✅ **Hermes 3** (полная собственная поддержка, все инструкции работают безупречно)
- ✅ **Claude (с MCP)** (незначительные изменения синтаксиса для вызовов инструментов; основная архитектура полностью применима)
- ✅ **Пользовательские GPT** (используйте файлы знаний для имитации памяти, Actions для имитации навыков)
- 🔸 Другие платформы агентов с **постоянная память + навыки / процедурные модули**
- 🔸 Платформы без функции Навыков (могут работать в ухудшенном режиме, см. [Глава 7 §7.1](chapters/07-faq-extensions.md))

## Основная концепция

> **Вы делаете не prompt engineering. Вы делаете architecture engineering.**
>
> Вы не учите агент новым знаниям. Вы используете мышление системной инженерии для проектирования внутренней архитектуры, которая работает стабильно и непрерывно эволюционирует.

Для получения дополнительной информации и пояснений см. [Пролог](PREFACE.md) и [Главу 1](chapters/01-background.md).

## Название проекта

**Cybernetic Your Agent** — внедрение кибернетических генов в ваш агент.

Мы не учим AI новым знаниям. Мы используем четыре концепции из Инженерной кибернетики Цянь Сюэсэня — предварительная связь, интегральное управление, иерархическое разделение и самоконтроль — для перестройки архитектуры памяти и навыков агента. Агент переходит от понимания команд к пониманию *вашей логики суждения* — по-настоящему становясь *вашим* агентом.
