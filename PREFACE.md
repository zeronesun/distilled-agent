# 序章：为什么 AI 学会了所有知识，却依然学不会“懂你”

你是否有过这样的经历：

你和一个 AI 助手已经合作了几十次。你反复告诉它“结尾不要总结升华”“代码先写测试再写实现”“方案先列风险再谈收益”。每次说，它都应承。每次新开一个会话，一切归零。下次你发同样的任务，它热情地又犯了一遍同样的错，等着你再一次纠正。

**问题不在于 AI 不够聪明。** 它的模型参数可能有数千亿。它读过人类有史以来几乎所有的公开文本。你告诉它的每一句话，它都“理解”。但它就是记不住——不是记不住信息本身，而是记不住**什么对你重要、什么在你这儿算是犯错、什么样的交付算合格**。

市面上解决这个问题的主流思路是：继续教。写更长的 prompt。整理更详细的规则文档。搞一套复杂的 RAG 知识库。每次 AI 犯错就再补一条规则进去，直到系统提示词膨胀到几万 token，规则之间开始互相矛盾，而你本人也记不清自己到底写过多少条要求。

**这本教程走一条相反的路。**

我们不教 AI 新知识。我们**用 20 世纪最硬核的系统工程理论，为 AI 重新设计记忆与技能的架构**。

这个理论的名字听起来很遥远——钱学森《工程控制论》。但你不需要是工程师，也不需要懂数学。你只需要理解四个概念，并按照本教程的步骤在你的 AI Agent 上部署：**前馈**（不等错误发生先避坑）、**积分控制**（吐槽一次不改，两次才系统性改）、**层级分离**（原则和步骤彻底分家）、**自监控**（交付前自动质检）。

这四个概念，加上一层周期性闭环反省，就能让你的 Agent 从“每次都要重新教的工具”，变成一个**主动避坑、不乱震荡、持续对齐你原则的协作引擎**。

这个思路不是坐在书斋里想出来的。

它始于一条笔记。在那条笔记里，一个用户尝试把钱学森工程控制论的核心思想“投喂”给他的 Hermes Agent，用来组织记忆体和技能层，让 Agent 工作时按这套逻辑触发对应能力。评论区一片“怎么搞”“试了后还真有用”。

这本教程，就是将那条笔记中的洞见，结合后续无数次的实测打磨和架构推演，**系统化为一套完整、可复制、面向通用 Agent 平台的实操方案**。我们从“蒸馏人的思考模式”这句话出发，一直走到能用控制论四概念驱动 Agent 自我进化的完整架构。

你不需要特定的模型。你不需要 Hermes。你需要的只是一个具备**持久化记忆**和**技能/程序性模块**的现代 Agent 平台，以及一份愿意用一两个小时搭建一套长期回报系统的耐心。

## 这本书不是什么

**这不是一本 prompt 合集。** 你在这里找不到“让 AI 写作更好的 50 条提示词”。我们关心的是架构，是系统，是怎么让提示词的组织方式本身变得可控和可进化。

**这不是一篇学术论文。** 我们对工程控制论的引用是高度工程化、高度简化的。钱学森的体系远比这里呈现的深广。我们只取了四个对 Agent 架构设计有直接指导价值的概念，并做了一件事：**把它们变成 Agent 可执行的指令。**

**这不是一次性改造。** 这套系统的核心特征是“闭环进化”——部署只是开始，不是终点。你每周说一次“起飞”，Agent 会用控制论标准自我审计，不断靠拢你的真实偏好。

## 这本书适合谁

如果你符合以下任何一条，这本书可能对你有用：

- 你有一个经常协作的 AI Agent，但每次新会话都像是在重新认识它。
- 你已经厌倦了反复纠正同一个问题，希望纠正一次就能沉淀为长期规则。
- 你对“把系统工程思想用在 AI 上”这件事本身感到好奇。
- 你不满足于把 AI 当成一个问答工具，想把它变成一个有稳定个性和自我纠偏能力的协作者。

如果你只是偶尔用 AI 聊聊天、问问一次性问题，这本书的架构对你来说可能太重了。你可以读前三章了解思想，后面的实操不必全量部署。

## 如何使用这本书

本书共七章，按照“为什么 → 是什么 → 怎么搭 → 怎么用 → 怎么修”的路径组织：

- **第一章到第四章**是理论基础和架构设计。如果你读完后觉得“这个思路我认可”，再进入实操。
- **第五章**是纯指令操作章。你可以直接复制粘贴指令发给你的 Agent，按步骤完成部署。
- **第六章**教你日常怎么用、怎么验证系统在正常运行。
- **第七章**是排障和延伸思考，碰到问题随时回查。

如果你使用的 Agent 平台是 Hermes（或类似的能力集合），可以几乎无缝执行全部步骤。如果你的平台能力有差异，第七章的 7.1 节提供了降阶适配方案。

## 开始之前，先说清楚一件事

部署这套系统需要你花一个小时。回报是：此后每一次协作，Agent 都会主动回忆你的偏好、主动避开已知的坑、在交付前自我质检，并在每周复盘时自动优化自己的行为。

**这不是让 AI 更“强”。是让它更“懂你”。**

两者之间的差距，就是一百条 prompt 和一个架构之间的差距。
