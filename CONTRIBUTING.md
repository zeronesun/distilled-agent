# 贡献指南

感谢你对 **Cybernetic Your Agent** 项目的关注。本项目的目标是让钱学森工程控制论的思想，能够被任何使用现代 AI Agent 的人轻松部署和受益。

欢迎任何形式的贡献。

---

## 贡献方向

以下是你可以为这个项目添砖加瓦的方式：

### 补充实践案例

在不同的 Agent 平台上部署本架构，并分享你的经验：
- 平台特有的适配方式（工具调用语法、记忆机制差异等）
- 实践中遇到的问题和解决方案
- 运行一段时间后的效果反馈（重复纠正率、前馈命中率等）

### 理论延展

引入新的理论框架来扩展 L1 元认知层：
- 例如：贝叶斯推理、香农信息论、TRIZ、约束理论等
- 要求：说明新理论如何与现有的四概念协同工作，并给出可执行的 L1 扩展条目

### 翻译

将教程翻译为其他语言。请先开 Issue 声明你要翻译的语言，避免重复工作。

### 文档改进

- 修正错别字、语法错误、表述不清
- 补充图表（架构图、数据流图）
- 改进示例文件

### 代码与脚本

- 为特定 Agent 平台提供自动部署脚本
- 提供记忆/技能状态导出的工具

---

## 提交规范

### Issue

- 提交 Bug、改进建议、新想法之前，请先搜索已有 Issue。
- 清晰描述问题或建议，附上相关上下文。

### Pull Request

1. **Fork 本仓库**。
2. 为新功能或修复**创建独立分支**（例如 `feat/add-claude-mcp-guide`）。
3. **保持提交信息简洁明确**（例如 `docs: 修正第三章积分控制阈值说明`）。
4. **关联对应 Issue**（如果有的话），在 PR 描述中使用 `Closes #issue号`。
5. 如果是新增内容，请**遵循现有文档的语气和结构**：
   - 简洁、直接、偏工程化
   - 有理论必有落地，有设计必有指令
6. **确保可以独立阅读**：每个文档文件应该是自包含的，不依赖上下文对话。

---

## 内容原则

本教程坚持以下原则，请贡献时务必遵守：

- **可执行优先于理论完备**：给出的内容必须是读者可以立刻操作的。
- **架构优先于堆量**：我们增加的是结构，不是 prompt 条目。
- **开放式结尾被允许**：不强求每个问题都有终极解，但要给出排查方向。

---

## 行为准则

- 讨论聚焦于教程本身，保持尊重和建设性。
- 欢迎不同意见的理性讨论，不欢迎人身攻击。
- 维护者保留关闭偏离主题或不遵循规范的 Issue/PR 的权利。
