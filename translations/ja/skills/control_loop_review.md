【制御ループ整理 — 根本原因分析版】
トリガー：「深い整理をして」または「離陸」

以下の9ステップで完全なレビューを実行：

第一ステップ：前フィードバック審査
- この期間内feedforward_startupが毎回正しく呼び出されたか？
- 無視された前フィードバック信号があるか？あれば記録し、次回の前フィードバックに組み入れる

第二ステップ：積分分降下実行
- [積分制御表]の完全な状態を読み込む
- 累積≥2回かつ「降下済み」が「否」の項目に対し、自動で降下を実行：
  → 対応するL2/L3/Skillに更新
  → 「降下済み」を「是」にマーク
  → 一つずつ確認

第三ステップ：階層浄化
- メモリをスキャン：操作手順が混入していないか？→ 対応するSkillに移動
- Skillをスキャン：原則内容が混入していないか？→ L2/L3に移動

第四ステップ：自検ゲートアップグレード評価
- delivery_gateのインターセプト記録を審査
- ゲート基準をアップグレードする必要があるか判断

第五ステップ：根本原因分析
問題現象 → 直接原因 → アーキテクチャ根本原因（規則ミスか実行ミスか）→ 更新決定 → 検証方法

第六ステップ：再利用可能な抽出
- この期間で新たなSkillとして抽象化できる経験を特定

第七ステップ：移行提案（オプション）
- 今回の体系的問題に基づき、新しい理論フレームワークを導入できるか判断

第八ステップ：ループ記録
今回のすべての結論をメモリに記録

第九ステップ：ユーザープロファイル維護リマインダー呼び出し
完了後にuser_profile_maintainerが自動的に呼び出されることを提示。
