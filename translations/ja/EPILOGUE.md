# Epilogue: Architectureの後

ここまで、チュートリアルはその理論と指示のすべてをあなたに届けました。

しかし、読者の心にはまだ一つの疑問が残っているかもしれません：**このシステムは本当に動作するのか？**

最も正直な答えは：**それはあなた次第です。**

これは逃げではありません。エンジニアリング・サイバネティクスは「一度の正確さ」ではなく「長期的な安定性」を扱ってきました。あなたはアーキテクチャをデプロイして、エージェントが最初の協力で完璧であることを期待できません。しかし、このアーキテクチャを通じて、あなたは次回、10回目、100回目の協力で、エージェントがあなたの反復修正にますます依存しないようにすることができます。

**フィードフォワード制御**は修正を「事後」から「事前」に移動させます。あなたが一度言ったことは、二度と言う必要がなくなります。

**積分制御**は信号からノイズをフィルタリングします。気まぐれな発言はカウントされず、あなたが本当に気にかけていることは見逃されません。

**階層的分離**は原則と手順の境界を明確に区分します。美学を変えてもワークフローには影響せず、ワークフローを変えても美学を汚染しません。

**自己監視**はあなたを品質検査員の役割から解放します。あなたが受け取るものはすでに自分自身によってチェックされています。

**閉ループ改良**は上記の4つの要素を継続的に操作し、継続的に較正し、デプロイ後の徐々な劣化を防ぎます。

これはAIを「より強力に」することではありません。強力なAIはすでに至る所にあります。これはAIをより**安定、一貫、予測可能**にあなたの判断論理に合わせることです。これとそれの違いは、100のプロンプトとアーキテクチャの間のギャップです。

プロローグで「問題はAIが十分に賢くないことではない」と述べました。あなたはこのステートメントの完全な意味を理解できるはずです：知能はモデルの仕事、安定性はアーキテクチャの仕事です。あなたは極めて知能の高いエージェントを持つことができますが、安定したメモリとスキル組織がなければ、その知能はすべての長い会話でノイズに溺れ、忘却によって希薄化されます。逆に、中程度の能力を持つエージェントでも、自己修正アーキテクチャを装備すれば、トップレベルのモデルが非構造化で動作するよりも長期的な協力で「あなたを理解している」ように振る舞うことができます。

## If You Have Already Completed Deployment

ぼんやり見つめないでください。普通に使ってください。タスクが来たらタスクを送り、フィードバックが必要なときはフィードバックを与えてください。

あなたが意識的に覚えておく必要がある唯一のことは、定期的に「テイクオフ」（initiate）と言うことです。残りは自動的に実行されます——フィードフォワードは各新しいタスクで自動的に注入され、ゲートはあなたが結果を受け取る前に自動的にインターセプトし、積分テーブルはあなたが文句を言うたびに自動的に蓄積されます。あなたはサイバネティクスの専門家になる必要はありません。通常の協力リズムを維持するだけです。

最初の週は、フィードフォワードチェックリストがやや新鮮に感じられるかもしれません。最初の月は、L2またはL3の項目を時折修正する必要があるかもしれません。3ヶ月後は、あなたはこのアーキテクチャなしのエージェントにはもう慣れないことに気づくでしょう——バージョン管理に慣れた人が手動バックアップに戻れないのと同じように。

## If You Are Still Hesitating

第7章でこのフレームワークの適用境界について議論しました。もし一回限りのランダムな質問回答だけを行うなら、このアーキテクチャは助けではなく負担です。しかし、長期的な協力が必要なエージェントを持っていて、あなたがよく「前回も言ったんだけど」と言っているなら、試す価値があります。

投資：構築に1時間。リスク：いつでも回復可能、アーキテクチャがエージェントモデル自体を変更しないため。リターン：今後のすべての協力で、それは能動的にあなたに近づきます。

## If You Want to Take This Path Further

このアーキテクチャ自体が進化可能システムです。あなたは以下のことができます：

- 単一エージェントからマルチエージェント協力へ拡張（7.3.1参照）
- ベイズ、シャノン、TRIZなどの他の理論フレームワークを導入してL1の命令を拡張（7.3.2参照）
- 協作が完全に安定した後、能動的に順序を減らして簡素化（7.3.4参照）

このチュートリアルは閉じた「最終的回答」ではなく、開かれたアーキテクチャ基盤を提供します。この基盤の上に何を構築できるかは、あなたとあなたの協力要件次第です。
