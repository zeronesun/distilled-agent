# Contributing

**Cybernetic Your Agent** プロジェクトにご関心を持っていただき、ありがとうございます。このプロジェクトは、銭学森の『エンジニアリング・サイバネティクス』のアイデアを、現代のAIエージェントを使用する誰もが簡単に導入・活用できるようにすることを目指しています。

あらゆる形の貢献を歓迎します。

---

## Contribution Areas

このプロジェクトに貢献できる方法は以下の通りです：

### Supplementing Practical Cases

異なるエージェントプラットフォームにこのアーキテクチャをデプロイし、経験を共有してください：
- プラットフォーム固有の適応方法（ツール呼び出し構文、メモリ機構の違いなど）
- 実践で遭遇した問題とその解決策
- しばらく運用した後の有効性フィードバック（修正反復率、フィードフォワードヒット率など）

### Theory Extensions

新しい理論フレームワークを導入してL1メタ認知レイヤーを拡張してください：
- 例：ベイズ推論、シャノン情報理論、TRIZ、制約理論など
- 要件：新しい理論が既存の4つの概念とどのように連携するかを説明し、実行可能なL1拡張エントリを提供してください

### Translation

このチュートリアルを他の言語に翻訳してください。重複作業を避けるため、翻訳する言語を宣言するIssueを先に開いてください。

### Documentation Improvements

- タイプ、文法エラー、不明確な表現の修正
- 図面（アーキテクチャ図、データフロー図）の補充
- 例ファイルの改善

### Code and Scripts

- 特定のエージェントプラットフォーム向けの自動デプロイスクリプトの提供
- メモリ/スキル状態エクスポートツールの提供

## Submission Guidelines

### Issue

- バグ、改善提案、新しいアイデアを提出する前に、既存のIssueを検索してください。
- 問題や提案を明確に説明し、関連するコンテキストを含めてください。

### Pull Request

1. **このリポジトリをForkしてください**。
2. **新しいブランチを作成して**、新機能や修正に取り組んでください（例：`feat/add-claude-mcp-guide`）。
3. **コミットメッセージを簡潔かつ明確にしてください**（例：`docs: fix integral control threshold explanation in chapter 3`）。
4. 対応するIssueがある場合は、PRの説明に`Closes #issue-number`を記載してリンクしてください。
5. 新しいコンテンツを追加する場合は、**既存のドキュメントのトーンと構造に従ってください**：
   - 簡潔、直接的、エンジニアリング指向
   - 理論があれば実装を、設計があれば指示を必ず含める
6. **独立した可読性を確保してください**：各ドキュメントファイルは自己完結しており、会話のコンテキストに依存しないようにしてください。

## Content Principles

このチュートリアルは以下の原則に従っています。貢献する際は確実に従ってください：

- **実行可能な理論完全性よりも**：提供されたコンテンツは読者がすぐに操作できるものである必要があります。
- **プロンプト量よりもアーキテクチャ**：プロンプトエントリではなく、構造を追加します。
- **オープンエンドの結末を許可する**：すべての問題に対する究極の解決策を要求しませんが、トラブルシューティングの方向性は提供する必要があります。

## Code of Conduct

- ディスカッションはチュートリアル自体に集中し、敬意と建設性を維持してください。
- 異なる意見の合理的な議論を歓迎しますが、個人攻撃は認めません。
- 管理者は、トピックから逸脱するか、ガイドラインに従わないIssue/PRを閉じる権利を留保します。
