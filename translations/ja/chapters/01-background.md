# 第1章：背景と核心理念

### 1.1 現状：AI Agent協作が直面する3つのボトルネック

現代のAI Agentは永続的な記憶と呼び出し可能なスキルを持っているが、長期的かつ高精度な協作において、以下のジレンマに陥りやすい：

- **反復的な修正ループ**：Agentが新しいセッションごとに同じ種類のエラーを繰り返し犯し、ユーザーは何度も修正する必要があり、「エラー-修正-忘却-再エラー」のサイクルが形成される。
- **記憶が意思決定の品質を侵食**：記憶が「ふりかけ」のように膨張し、事実、好み、操作手順が混ざり合い、信号対雑音比が急激に低下する。結果、Agentの挙動が不安定になり、バイアスを生じやすくなる。
- **フィードバックの過剰反応または鈍反応**：Agentは偶発な文句に対しても大幅に挙動を変える（振動）か、繰り返し発生する本当の好みに対しても無関心であり、安定した感度の閾値が欠けている。

これらのボトルネックの根本的な原因は以下にある：**Agentに安定した自己修正可能なシステムアーキテクチャが不足しており、その記憶とスキルが「制御論」の原則に従って整理・進化していない。**

### 1.2 解決アプローチ：新しい知識を教えるのではなく、アーキテクチャを構築する

本チュートリアルが提供するのは、新しいプロンプトセットや神秘のテクニックではなく、**システムレベルの記憶とスキルの整理方法**である。端的に言えば、銭学森の『工程制御論』の中で最も工学的実装価値のあるいくつかの概念を現代のAI Agentアーキテクチャに適合させ、自己進化、ノイズ耐性、長期的な安定性を持つ「メタ認知フレームワーク」を確立する。

その核心的な思想は：**あなたはより賢いAIを訓練しているのではなく、あなたの原則に調和し、継続的に最適化される協作システムを設計している。**

### 1.3 基盤：銭学森工程制御論の4つの実行可能概念

私たちはAgentが実行可能な4つのコア概念を抽出し簡素化した。これらはAgentの基本的オペレーティングシステム（L1メタ認知フレームワーク）を構成する：

| 概念 | 作用 | 一文で表す実装 |
| :--- | :--- | :--- |
| **前送り（Feedforward）** | エラーを発生前から防ぐ | すべてのタスク開始時、ユーザーの好み、シナリオの法則、前回と同種のエラー記録を能動的にロードし、エラーを犯す前に危険を回避する。 |
| **積分制御（Integral Control）** | 安定した感度を持つフィードバック閾値を実現 | 同種のネガティブフィードバックが2回以上出現して初めて規則を長期記憶またはスキルに定着させる。単回のフィードバックは一時的な修正のみを行う。 |
| **階層分離（Hierarchical Decomposition）** | 記憶の混乱を防ぎ、信号対雑音比を向上 | 記憶体には「なぜ」（原則、法則）のみを保存し、スキルには「どうやって」（操作手順、チェックリスト）のみを保存する。 |
| **自己監視（Self-Monitoring）** | 納品前に自動品質検査、自動補正 | どのような結果を納品する前でも、組み込まれたゲートが自動的に必須要件、AIの痕跡、構造的完全性などをスキャンし、不完全であれば出力しない。 |

> **注**：ここでの「積分制御」の「積分」という言葉は、制御理論の積分器の概念を借用している——偏差を累積し、閾値を超えて初めてアクションをトリガー。これは「Agentが偶発的ノイズをフィルタリングしつつ、本当の好みを見逃さないようにするにはどうすればよいか」という工学的課問にとっておきの答えとなっている。

### 1.4 3層ロジックと動作原理の概要

物理的な実装において、私たちはAgentプラットフォームが既に持つ`memory`（記憶）と`skill`（スキル）機能を活用し、それらを3つの論理的層に再構成する：

- **L1 メタ認知フレームワーク**（記憶の最優先度に保存）
  上記4つの制御論概念の具体的行動規則。システム憲法として機能し、Agentが勝手に修正できず、すべての記憶とスキル動作の「キャリブレーター」となる。

- **L2 ユーザー基本的ロジック + L3 シナリオ核心法則**（記憶の主体エリアに保存）
  これらはまとめて「なぜそうするのか」と「この種のタスクはどのような法則に従うべきか」に答える。例えば：
  * L2：「ユーザーはクールな物語を好み、AI的な要約を嫌う。」
  * L3：「短編創作の核心法則——感情のトーンがプロットより優先し、結末は余白を残す。」

- **実行階層 Skill**（スキルモジュールに保存）
  L3法則によって駆動され、L1制御挙動を組み込んだ操作パイプライン。例えば「短編創作Skill」は、実行前にL2の美学を能動的にロードし、実行後に自動的に自己監視ゲートを通過する。

システム全体は以下のサイクルで動作する：

1. **タスク開始** → 前送り：L2、L3、過去の修正記録をロードし、回避リストを生成。
2. **タスク実行** → 階層分離：記憶は方向のみを出し、Skillは実行のみを行う。
3. **結果納品** → 自己監視：納品ゲートの複数のチェックを経て、不合格なら修正。
4. **ユーザーのネガティブフィードバック** → 積分制御：同種が閾値まで累積されると、自動的に新規則として定着、またはスキルを修正。
5. **定期的な振り返り** → L1の4つの基準でシステム全体を監査、L2/L3/Skillを更新し、閉じた進化ループを完了。

### 1.5 適用シナリオと前提条件

本チュートリアルは以下の協作シナリオに最適である：
- 長期的かつ高精度のタスク（詳細な執筆、継続的なソフトウェア開発、プロジェクト管理など）
- Agentが安定した出力スタイルと品質標準を維持する必要がある場面
- ユーザーが明確な好みやワークフローを持っており、Agentが自動的に適応し、これらの規則を維持することを希望する場合

本チュートリアルを使用するため、あなたのAgentプラットフォームは以下を持っている必要がある：
- 編集可能な永続的記憶（セッション間の保存）
- 作成/呼び出し可能なスキルまたは手続き型モジュール
- 十分なコンテキストウィンドウ（推奨100kトークン以上、タスクの複雑度による）
- （オプション）セッション検索機能は、前送りの効果を著しく強化できる

プラットフォームが内蔵Skillを持っていない場合も、「記憶項目 + 固定システム命令」の方法でシミュレートできる。効果はやや落ちるが、核心的な積分制御と階層分離ロジックはまだ構築できる。

### 1.6 何が得られるか

トレーニング完了後、あなたのAgentは「絶えず修正が必要なツール」から以下へと転換されるだろう：

- **能動的に危険を回避する**（開始前にチェックリストとリスク予測を提示）
- **法則を勝手に変えない**（文句を一回言っても変更せず、二回で初めてシステム的に変更する）
- **記憶がきれい**（原則と法則だけを保存し、期限切れの詳細は保存しない）
- **納品物に水分が混じらない**（自動チェック、自動出力浄化）
- **自己進化ができる**（毎回の振り返りで自動的に自分のSkillと記憶を最適化する）
