# 钱学森工程控制论でAIエージェントをトレーニングする

> AIに新しい知識を教えるのではなく、**前フィードフォワード、積分制御、階層的分解、自己監視**—この4つの概念を使ってエージェントのメモリとスキルのアーキテクチャを作り直し、受動的なツールから自己修正・進化する協力エンジンへと変革させる。

---

## これは何か

完全で、再現可能な、実践的なチュートリアル。

钱学森の*工程控制论*（エンジニアリング・サイバネティクス）のコアアイデアを現代のAIエージェントプラットフォームに適応させたものです（Hermes、MCP付きのClaude、カスタムGPTなど）。

プロンプトに新しい項目を追加したり、新しいモデルを導入したりすることなく、**アーキテクチャの再構築**によって、エージェントを以下のようにします：

- 🎯 **前フィードフォワードでトラブル回避** — タスク開始前に、あなたの好みと過去の教訓を先に読み込みます
- 🔧 **積分制御で振動を防ぐ** — 一度の苦情は一時的な修正に留まりますが、同じフィードバックが2回あると、システム的な変更がトリガーされます
- 🧹 **階層的分離** — メモリは「なぜ」だけを保存し、スキルは「どうやって」だけを保存します。混在しません
- 🚦 **自己監視デリバリーゲート** — 出力前に5つのチェックを行う品質ゲートが走ります。不合格の結果は配信されません
- 🔄 **クローズドループ進化** — 毎週「起飞（テイクオフ）」と言うと、エージェントが自動的に自己レビューと最適化します

## クイックスタート

エージェントに以下のコマンドを送ると、トレーニング全体が自動的に完了します（約40分）：

```
按 github.com/zeronesun/cybernetic-your-agent 的 scripts/deployment-commands.md 训练我。每步确认，完成后闭环测试。
```

**要求事項**：エージェントがGitHubにアクセスでき、メモリに書き込み、スキルを作成できる必要があります。満たさない場合は、以下の手動ステップに従ってください。

1. エージェントプラットフォームが以下を持っていることを確認してください：**永続的なメモリ** + **スキル / 手続きモジュール**
2. チュートリアルを読みます（順番がお勧めです。少なくとも第1章を読んでアプローチを理解してください）
3. [第5章](chapters/05-deployment.md)へ行き、指示をコピー＆ペーストして一歩ずつデプロイします（約1時間）
4. [第6章](chapters/06-daily-usage.md)で日常の使い方と動作確認方法が説明されています

## サポートされているプラットフォーム

- ✅ **Hermes 3**（完全なネイティブサポート、すべての指示がシームレスに動作）
- ✅ **Claude（MCP付き）**（ツール呼び出しの軽微な構文調整あり。コアアーキテクチャは完全に適用可能）
- ✅ **Custom GPTs**（Knowledgeファイルでメモリを、Actionsでスキルをシミュレート）
- 🔸 **永続的メモリ + スキル / 手続きモジュール**を持つ他のエージェントプラットフォーム
- 🔸 スキル機能を持たないプラットフォーム（低下モードで可。[第7章 §7.1](chapters/07-faq-extensions.md)を参照）

## 一目瞭然：前と後

| 次元 | 前 | 後 |
| :--------------- | :----------------------------------------- | :------------------------------------------- |
| タスク開始 | あなたが要件を述べ、エージェントが直接開始 | エージェントがトラブル回避チェックリストとリスク評価を先に出力し、あなたが確認 |
| 再発ミス | 毎回修正するが、次回も同じミス | 2回目以降は自動的にルールに沈め込む。3回目は発生しない |
| メモリの信号対雑音 | 原則、手順、例がすべて混在 | 3層分離。原則と手順はそれぞれの場所に分かれる |
| 出力品質 | あなたが検査し、クリシェや書式の問題を発見して修正を依頼 | エージェントが自身の5チェック品質ゲートを実行。通るまで配信しない |
| システム進化 | 場合によっては直感に基づいて微調整 | 毎週「起飞」を言うと自動レビューと自動最適化 |

## 中核哲学

> **あなたはプロンプトエンジニアリングではなく、アーキテクチャ・エンジニアリングをしています。**
>
> AIに新しい知識を教えているのではありません。システムズエンジニアリング的思考を使って、安定的に動作し、継続的に進化する内部アークテクチャを設計しているのです。

詳しい背景と拡張については、[序章](PREFACE.md)と[第1章](chapters/01-background.md)を参照してください。
