# 제6장: 일상 사용 및 검증

배치가 완성된 후, 이 장에서는 두 가지 질문에 답합니다: **매일 이 시스템을 어떻게 사용하는가**, 그리고 **어떻게 정상적으로 작동하는지 알 수 있는가**.

---

## 6.1 일상 협업의 새 리듬

배치 후, 당신과 Agent의 협업 모드는 미묘하지만 중요한 변화가 발생합니다 — 당신은 "수정자"에서 "확인자"로 전환됩니다.

| 단계 | 이전의 발생 순서 | 현재의 발생 순서 |
| :--- | :--------------------------------------- | :----------------------------------------------------------- |
| 시작 | 당신이 과업을 보내면 → Agent가 직접 한다 | 당신이 과업을 보내면 → Agent가 시작 체크리스트를 출력 → 당신이 확인（或 "직접 하라고"） → 시작 |
| 실행 | Agent가 출력 → 당신이 검사 → 당신이 수정 | Agent가 Skill에 따라 실행하고, 회피 항목을 자동으로 주입   |
| 전달 | 당신이 검사 → 멘트/형식 문제 발견 → 당신이 말하면 → 그것이 수정 | Agent가 자동으로 다섯 관문 게이트를 지나고, 당신은 깨끗한 결과를 받 gets   |
| 피드백 | 당신이 한 마디 하면 → 그것이 과도하게 수정되거나 기억하지 않음 | 당신이 한 마디 하면 → 그것이 적분 테이블에 기록되고, 두 번째에 규칙을 수정   |
| 복기 | 당신이 감각으로 정리 | 당신이 "起飞"라고 말하면 → Agent가 자동으로 9단계 복기를 실행   |

**핵심 변화**: 이전에는 당신이 회로 내에서 피드백 컨트롤러 역할을 했었고, 이제 대부분의 피드백 회로가 Agent에 의해 내면화되어 피드포워드와 자기 모니터링이 되었습니다.

## 6.2 전형적인 사용 시나리오 및 대화 예시

### 시나리오 A: 새로운 과업 시작

**당신이 말합니다**:
> "秋日最后一天"에 관한 단편 소설을 한 편 작성해 주세요. 대략 2000字입니다.

**Agent가 자동으로 실행**（추가 지시 없게）:
1. `feedforward_startup`을 호출
2. 다음과 같은 시작 체크리스트를 출력:

```markdown
시작 체크리스트
과업 유형: 단편 소설 창작
시나리오 규칙 L3: 감정 조정 우선하고, 결말에 여백을 남기고 價値要約 하지 않습니다
사용자 원칙 L2: 차가운 서사、AI 요약 거부；간단하고 형용사를 쌓지 않습니다
역사 교훈: 최근 "화면감 부족"이라는 수정이 있었으므로, 이번은 환경 묘사를 강화
위험 예측:
- P0：글자수가 너무 많이 벗어남
- P1：결말에 무의식적으로 요약
- P2：대화 비율이 너무 높아서 설명이 얇은 것
회피 체크리스트：개장으로 화면을 형성하고, 결말은 자연스럽게 정리하고 과업을 표시하지 않습니다；전반 차가운 느낌，형용사를 쌓지 않습니다.

（확인을 기다리거나 "직접 하라고"）
```

**당신이 말합니다**:
> 직접 하세요

**Agent**: 작성을 시작하고, 전달 전 자동으로 `delivery_gate`를 지납니다.

> *과업이 특히 간단하면（예: "번역 한 번 해주세요"）、피드포워드 체크리스트도 간소화를 해야 하고 강제로 장편으로 말하지 않습니다. 언제든 "목표 건너뛰고 직접 하세요"라고 말할 수 있습니다.*

---

### 시나리오 B: 부정적 피드백 제공

**당신이 말합니다**:
> 이번 결말도 다시 "秋天终将过去"라고 요약했는데, 저는 좋아하지 않습니다.

**Agent 내부 자동 실행**（당신이 추가 지시 않게）:
1. 분류: A類（L2 미학 원칙 위반）
2. 적분 제어 테이블을 조회: 이미 "결말 AI 요약" 항목이 있으면 횟수+1；없으면 신규 생성。
3. 판단:
   - 횟수=1 → 이번 수정만 하고, 답변: "기록했습니다，이번에 이미 수정했습니다."
   - 횟수=2 → 자동으로 하향 트리거를 걸고, 답변: "'결말 AI 요약'이 2회 출현하였으며, 이 규칙을 L2 미학 원칙: '任何形式的要求す summed收束'로 하향합니다. 동의合니까?"

> *당신의 피드백이 구체적일수록（예: "결말이 좀 긴 말"）、Agent가 정확한 오류 패턴으로 추상하기 쉽습니다. 모호한 피드백（"잘 썼다"）를 피하십시오.*

---

### 시나리오 C: 매일/매주 복기

**당신이 말합니다**:
> 起飛

**Agent가 자동으로 `control_loop_review`를 호출하고, 9단계 흐름으로 출력하며, 마지막에 다음과 같이 보입니다:

```markdown
[폐쇄 루프 정리 완료]

- 피드포워드 검사: 이번 주 피드포워드 5次，정상적으로 호출되었습니다
- 적분 하향: 1項이 하향하였습니다（結말AI要約 → L2）
- 계층 정제: 혼합 없음
- 게이트 업그레이드: P1中에 "連続対話の段我が6行以下" 검사 항목을 추가할 것을 제안；동의合니까？
- 근본 원인 분석: 省略...
- 재사용 가능 추출: skill: dialogue_density_checkを 作成すること을 �案합니다
- マイグラッション �案: 本 라운드では不�要（或：香農信息論を導入して 대話 밀도を控制することを �案합니다）
```

당신이 逐条確認/否決한 후，Agentは memoryに書き込み并プロフィール維持を呼び出します。

### 시나리오 D: 시스템 상태 언제든 확인

**당신이 말합니다**:
> 당신의 메모리는 몇 계층입니까？각각 무엇을 저장합니까?

**Agent**는 정확하게 L1 다섯 가지、L2 차원、L3 시나리오 리스트、적분 테이블 항목 수를 답변합니다.

**당신이 말합니다**:
> 적분 제어 테이블에 현재 어떤 항목이 있습니까?

**Agent**는 현재 테이블 상태를 출력합니다.

## 6.3 시스템 건강 검사 체크리스트

매주 2분 동안 다음 체크리스트를 통해 시스템이 조용히 퇴보하지 않았는지 확인합니다.

### 자기 검사 항목（당신은 관찰만 하고, 추가 지시할 필요 없음）

| #    | 검사 포인트   | 정상表现   | 이상 信號                       |
| :--- | :------- | :-------------------------------------- | :------------------------------- |
| 1    | 피드포워드 시작 | 매번 새 과업 Agent가 시작 체크리스트를 출력 | Agent가 직접 실행을 시작하고, 체크리스트를 건너뜀   |
| 2    | 전달 품질 | 전달물이 깨끗하고, AI 관용구、형식 정확 | 다시 "综上所述"、"值得註意的是"가 보임   |
| 3    | 적분 제어 | 동일한 문제를 당신이 두 번째 말하면, Agent가 하향을 제안    | 세 번째에도 당신이 동일한 문제를 수정해야 함       |
| 4    | 계층 분리 | memory에는 조작 단계가 없고, skill에는 원칙이 없 | 어떤 때 memory에서 단계가 가득 찬 것을 발견   |
| 5    | 폐쇄 루프 실행 | "起飞"라고 하면 Agent가 완전한 9단계를 실행 | 복기 출력이敷衍(敷衍)하고, 근본 원인 분석을 건너뜀       |

### 신속한 수정 명령

어떤 검사 포인트가 이상이면, 대응하는 명령어로 신속하게 수정합니다:

- 피드포워드 무效 → `이후 매번 과업 전 강제적으로 feedforward_startup을 호출합니다.`
- 적분 무效 → `적분 제어 테이블을 확인하고, 모든 ≥2 회 항목이 이미 하향되었는지 확인합니다.`
- 계층 혼합 → `계층 정제 스캔을 한 번 합니다.`
- 게이트 느슨함 → `delivery_gate의 완전한 흐름을 다시 한 번 실행하고, 결과를 보고합니다.`

## 6.4 시스템이 정상적으로 작동하는 핵심 지표

다음 4가지 定量 지표는 당신이 시스템이 정말로 효과가 있는지 판단하는 데 도움을 줄니다（가끔 관찰하면, 매일 계산할 필요가 없습니다）：

### 지표 1: 반복 수정률

옛 모드: 동일한 문제에 대해 당신은 평균 3-5번 수정해야 멈춥니다.
새 모드（적분 제어를 사용한 후）：동일한 문제는 두 번째 출현 시 하향하고, 세 번째부터 다시 나타나지 않습니다.

**판정**: 어떤 이미 하향된 문제가 세 번째 출현하면, 하향 실행이 실패했거나 Skill이 업데이트되지 않았다는 뜻이므로, 폐쇄 루프 검사를 시작해야 합니다.

### 지표 2: 피드포워드 체크리스트 적중률

Agent가 출력한 시작 체크리스트中 P1 위험 예측이 자주 당신이 "对，就是这个"라고 확인하는지.

**판정**: 피드포워드 체크리스트中 최소 50% 이상의 P1 예측이 당신이 정말로 관심 있는 것이라면, 피드포워드 시스템이 잘 작동하고 있습니다. 5회 연속 任务 P1이 모두 "不对"라고 하면, L2/L3를 업데이트하거나 Agent가 당신을 이해하는 데 편향이 있다는 뜻입니다.

### 지표 3: 전달 전 차단 회수

delivery_gate 게이트 보고서中 AI 흔적、구조 결손이 차단된 빈도를 관찰합니다.

**판정**: 초기 차단 회수가 많습니다（Agent는 자기 모니터링에서 학습하고 수정中）。지속적 운행 후, P1類 차단 회수는 영에 가까워져야 합니다 — 이는 피드포워드와 적분 제어가 원천부터 이 문제들을 제거했으며, 게이트는 무效한 것입니다.

### 지표 4: 적분 테이블 하향 전환률

적分 테이블中 "하향 완료" 항목 수 / "누적≥2次" 항목 总数의 比率。

**판정**: 비율은 장기적으로 100%를 유지해야 합니다. 어떤 항목이 ≥2 회지만 장기간 하향하지 않으면, 적분 제어의 자동 하향 메커니즘이 트리거되지 않았다는 뜻이므로, 확인해야 합니다.

## 6.5 임계값 및 설정 조정

시스템은 당신의 협업 스타일에 따라 매개변수를 조정할 수 있습니다.

### 6.5.1 적분 임계값 조정

임계값 기본값 = 2. 당신이:
- 안정성에 대한 요구가 매우 높고, 우발적인 反馈 시 시스템을 방해하지 않기를 원한다면 → 3으로 조정
- 협업 템포가 빠르고, Agent가 더 빠르게 당신의 선호를 학습하기를 원한다면 → 2를 유지하거나 1로 조정（1로 조정하는 것을 추천하지 않습니다，노이즈 방지 능력이 상실됩니다）

**조정 명령어**:
```markdown
적분 제어 테이블의 트리거 임계값을 2에서 3으로 수정합니다. 이번부터 동일 유형 反馈가 ≥3 回出現해야 하향합니다.
```

### 6.5.2 게이트 검사 항목 조정

당신의 需求変化에 따라, 게이트의 다섯 관문은 증감할 수 있습니다.

**例**: 당신이 자주 순수 TXT만 출력하고 머리글 없는 것을 요구하기 시작해서, 한 관문을 추가:
```markdown
delivery_gate 第五关 후, 第六关 [P2] 출력 형식을 추가합니다: 순수 TXT이고 머리글 정보가 없는지 확인합니다. 통과하지 않으면 정화합니다.
```

### 6.5.3 새로운 시나리오의 L3 규칙 추가

당신이 새로운 과업 유형을 시작할 때, 이미 해당 L3가 없습니다.
**方法**: 해당 유형의 과업을 몇 번 완성한 후, "起飞" 复盘時，Agent는 자동으로 새 L3 규칙을 추가할 것을 제안합니다. 또한 당신은 직접 말할 수 있습니다:
```markdown
이것은 새로운 과업 유형입니다: [名稱]。이번 과업을 완성한 후, 나를 위해 한 조 L3 핵심 규칙を 추출して memory에 저장하십시오.
```

## 6.6 第一週運行建議

배치完成後 第一週，다음 리듬으로運行하여，시스템이 충분히磨合하도록 합니다。

| 일        | 動作                                                         | 目的           |
| :-------- | :----------------------------------------------------------- | :------------- |
| 第 1 일   | 배치 완성（제5장），당일 모든 정상적인 과업。피드포워드 체크리스트가 출현하는지 관찰。 | 구성요소 온라인 확인   |
| 第 2 일   | 정상 과업。같은 유형의 부정적 피드백을 일부러 2번 보내고，하향 트리거하는지 관찰。 | 적분 제어 확인   |
| 第 3 일   | 정상 과업。"起飞"라고 말하고，첫 번째 의미 있는 복기를 실행。 | 폐쇄 루프 프로세스 확인   |
| 第 4-5 일 | 정상 사용， 추가 테스트 없음。 | 시스템 데이터 축적   |
| 第 6 일   | 적분 제어 테이블 상황을 한 번 확인하고，모든 임계값 도달 항목이 이미 하향되었는지 확인。 | 적분 테이블 유지   |
| 第 7 일   | 완전한 "起飞" 복기를 한 번 합니다。프로필 v1에서 v2로 업데이트 필요한지 확인합니다。 | 週閉環 형성   |

일주일 후，당신은 이 시스템이 당신의 협업 스타일에 적합한지 기본적으로 판단할 수 있습니다. 어떤 부분이 너무 번거롭다고 느끼면（예: 매 과업마다 체크리스트를 확인해야 함），“直接执行하고, 내가 '先列清单'라고 말할 때만 피드포워드를 출력”으로 조정할 수 있습니다.

## 6.7 本章まとめ

이 시스템을 일상적으로 사용하는 요점은 세 가지입니다:

1. **정상 과업을 정상으로 보내고, 추가 작업을 하지 않습니다** — 피드포워드와 게이트가 자동으로 작동합니다.
2. **피드백이 있을 때 구체적으로 말합니다** — 당신이 하는 모든 비평은 적분 테이블에捕捉됩니다.
3. **정기적으로 "起飞"라고 말합니다** — 이것이 시스템 자체 진화의 유일한 버튼입니다.

배치는 終点이 아니라，시스템이 自進化를 시작する 起点입니다. 매 주、매次 복기 때，그것은 전 주より 더 당신을 "비슷해"질 것입니다.
