# L2/L3 초기 출력 예시

이 파일은 전형적인 L2/L3 초기화 대화를 보여줍니다. 사용자가 정제 명령을 보낸 후, Agent는 사용자의 이력에 대한 이해를 바탕으로 구조화된 L2와 L3 초안을 출력합니다.

> **참고**: 이 예시의 사용자 선호도는 가상이며, 출력 형식과 정제 세분도를 보여주기 위한 용도입니다. 당신의 Agent가 당신과의 이력 대화를 바탕으로 너에게 맞는 버전을 생성할 것입니다.

---

## Agent의 정제 출력

나의 현재 당신에 대한 이해를 바탕으로 정제 결과입니다:

### [L2·사용자 기본 논리]

- **소통 선호**: 간결 직접, 예인 말은 생략; 경계 명확, "아니오"라고 하면 즉시 멈춘다
- **품질 표준**: 코드 우선 검증 방법 포함; 출력은 재사용 가능해야 하며, 한 번 올바르면 충분하지 않음
- **결정 모드**: 리스크 우선, 먼저 완전한 문제를 드러낸 후 방안을 논의; 이차적 평가가 직관보다 우선
- **학습 모드**: 분야 이론 이동; 지행합일, 이해는 반드시 실시 가능한 실행으로

### [L3·상황 핵심 규칙]

- **단편 창작**:
  - 감정 정조가 줄거리 전개보다 우선
  - 끝은 여백으로, 가치 요약이나 도덕적 승급 없음
  - 백묘으로 화면감 구축, 비유에 의존하지 않음

- **코드 출력**:
  - 먼저 테스트나 검증 방법을 쓴 후, 구현을 씀
  - 기능 완전성이 주석 상세함보다 우선
  - 불확실한 동작은 "검증 필요"로 표시, 강제 작성 없음

- **아키텍처 설계**:
  - 먼저 시스템 경계와 데이터 흐름을 정의한 후, 기술 스택을 선택
  - 확장 가능성과 실패 모드 평가, happy path만 쓰지 않음
  - 출력은 리스크 분류(P0-P3) 및 대응 포함

- **문제 조사**:
  - 먼저 로그/실행 현장 근본 원인을 위치 지정, 경험으로 추측하지 않음
  - 불확실한 곳은 "검증 필요"로 표시
  - 재사용 가능한 조사 단계를 제시, 결론만 제시하지 않음

---

## 사용자의 수행 가능한 검토 동작

1. **항목별 확인**: 각 원칙마다 스스로 묻습니다 — "이것이 정말로 나의 안정된 선호인가?"
2. **편차 수정**: Agent에게 직접 어디가 틀렸는지 말하고, 어떻게 고쳐야 할지 말하세요
3. **누락 보완**: 특정 차원/상황이 핵심 원칙을 누락했다고 느끼면 직접 말하세요
4. **사실 아닌 것 삭제**: Agent가 강제로 만든(당신에게 이 선호가 없는) 항목은 직접 삭제하세요

## 일반적인 수정 예시

| Agent의 원본 정제 | 사용자의 수정                    | 설명                                                    |
| :---------------- | :-------------------------------- | :------------------------------------------------------ |
| 소통 선호: 간결 직접 | 소통 선호: 간결하되 충분한 컨텍스트 필요 | 사용자가 컨텍스트 필요함을 발견, 짧은 것만으로 부족       |
| 단편: 끝 여백     | 단편: 끝 여백; 하지만 시리즈는 온화한 결합 허용 | 사용자가 예외 보완                                        |
| 코드 출력: 테스트 우선 | 코드 출력: 빠른 프로토타입일 때 테스트 후追加 | 사용자가 프로토타입과正式输出的不同标准区分            |

## 쓰기 명령

검토 후 다음을 보냅니다:
> 위에서 확인한 L2와 L3를 메모리에 쓰십시오.

Agent는 그것을 메모리의 주 영역에 저장하며, [L2·사용자 기본 논리]와 [L3·상황 핵심 규칙]로 분산된 노드로 저장합니다.
