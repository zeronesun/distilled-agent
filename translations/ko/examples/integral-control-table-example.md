# 적분 제어 표 예시

이 파일은 한 달 운영 후 적분 제어 표의 실제 상태와 전형적인 하향 대화를 보여줍니다.

> **참고**: 아래 항목은 가상이며, 데이터 구조와 운영 로직을 보여주기 위한 용도입니다.

---

## 한 달 운영 후 적분 제어 표

```
【적분 제어표】
| 유형 | 오류 패턴(≤8자) | 첫 발생일   | 누적 횟수 | 임계값 | 하향 완료 | 하향 대상                            |
| ---- | --------------- | ---------- | -------- | ---- | -------- | ----------------------------------- |
| A유형 | 끝 AI 요약        | 2025-05-10 | 2/2      | 2    | 예        | L2·미학 : 요약식 결합 거부           |
| A유형 | 어조가 너무 열정적 | 2025-05-12 | 1/2      | 2    | 아니오    | -                                   |
| B유형 | 단편 화면감 부족  | 2025-05-10 | 2/2      | 2    | 예        | L3·단편 : 도입부 백묘으로 화면 정립  |
| B유형 | 코드 테스트 미포함| 2025-05-15 | 1/2      | 2    | 아니오    | -                                   |
| C유형 | 게이트 AI 관용어 누락| 2025-05-10| 2/2      | 2    | 예        | delivery_gate : P1 관용어 검출 규칙 업그레이드|
| C유형 | 피드백 예산 검색 누락| 2025-05-18| 1/2      | 2    | 아니오    | -                                   |
| D유형 | 적절하지 않은 비유| 2025-05-08 | 1        | -    | -         | (우연한 발생, 누적하지 않음)         |
```

## 하향 트리거 예시

### 시나리오: 사용자가 두 번째로 "끝 AI 요약" 지적

**사용자**:
> 이번도 끝에 "어떻든 살은 계속된다"라고 썼는데, 이런 요약은 싫어요.

**Agent 내부 과정**:
1. 오류 분류 → A유형(L2 미학 원칙 위반)
2. 적분 표 검색 → "끝 AI 요약" 찾음, 누적 횟수 1→2로 변경
3. 횟수 = 임계값 2, 하향 트리거

**Agent 출력**:
> 기록했습니다. "끝 AI 요약"이 2회 발생했습니다. 이 규칙을 **[L2·사용자 기본 논리] 미학적 성향**으로 보내고, 항목 추가를 제안합니다: **"모든 형태의 요약식 결합 거부"**.
>
> 동의하십니까?

**사용자**:
> 동의합니다.

**Agent 실행**:
1. L2에 해당 항목 기록
2. 적분 표 업데이트: 하향 완료 → 예
3. 확인: "하향 완료했습니다. 향후 작업에서 피드백 스타트업 시 이 원칙을 능동적으로 불러오겠습니다."

## 각 단계 폐쇄 위원 정리 중 적분 표 보고 예시

> 이 단계 적분 제어표는 총 7개의 기록이 있습니다:
> - 하향 완료 3개(끝 AI 요약, 화면감 부족, 게이트 AI 관용어 누락), 추가 조치 불필요
> - 임계값 미달 3개(어조 열정적, 코드 테스트 미포함, 피드백 예산 검색 누락), 모두 1/2 회, 관찰 계속
> - 우연한 이벤트 1개(적절하지 않은 비유), 누적 시스템에 미포함
>
> 현재 하향 전환율: 100%(임계값 도달 모두 하향 완료)

---

## 적분 표가 정상 작동하는지 확인하는 방법

다음 신호를 관찰하세요:
- 매번 부정 피드백 후, Agent가 명확히 분류하고 누적하는지
- 누적 2회 도달 항목에 대해 Agent가 능동적으로 하향 제안하는지
- 하향 완료 후, 유형 문제가 다시 발생하지 않는지
- 매월 폐쇄 위원 정리에서 하향 완료 항목이 명확히 추적 가능한지
