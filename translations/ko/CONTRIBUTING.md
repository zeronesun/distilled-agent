# 기여 가이드

**Cybernetic Your Agent** 프로젝트에 관심을 주셔서 감사합니다. 이 프로젝트의 목표는 첸쉐썬의 공학 제어론 사상을 현대 AI 에이전트를 사용하는 모든 사용자가 쉽게 배치하고 혜택을 받을 수 있도록 하는 것입니다.

어떤 형태의 기여든 환영합니다.

---

## 기여 방향

프로젝트에 다음과 같이 기여할 수 있습니다:

### 실용 사례 추가

다양한 에이전트 플랫폼에 이 아키텍처를 배치하고 경험 공유:
- 플랫폼별 특화 어댑터 방식 (도구 호출 구문, 기억 메커니즘 차이 등)
- 실습 중 문제와 해결 방안
- 일정 시간 실행 후 피드백 (반복 수정률, 전위 피드백 적중률 등)

### 이론 확장

L1 메타인지 층을 확장하기 위해 새로운 이론 프레임워크 도입:
- 예: 베이지안 추론, 섀넌 정보론, TRIZ, 제약 이론 등
- 요구사항: 새로운 이론이 기존 4개념과 어떻게 협력하는지 설명하고 실행 가능한 L1 확장 항목 제시

### 번역

튜토리얼을 다른 언어로 번역. 중복 작업을 피하기 위해 먼저 번역할 언어를 이슈로 선언할 것.

### 문서 개선

- 맞춤법, 문법, 표현 불명확 수정
- 다이어그램 추가 (아키텍처 다이어그램, 데이터 흐름 다이어그램)
- 예시 파일 개선

### 코드와 스크립트

- 특정 에이전트 플랫폼을 위한 자동 배치 스크립트 제공
- 기억/스킬 상태 내보내기 도구 제공

## 제출 규범

### Issue

- 버그, 개선 제안, 새로운 아이디어 제출 전 먼저 기존 이슈 검색.
- 문제나 제안을 명확하게 설명하고 관련 컨텍스트 첨부.

### Pull Request

1. **이 저장소 포크**.
2. 새로운 기능이나 수정을 위해 **독립 브랜치 생성** (예: `feat/add-claude-mcp-guide`).
3. **커밋 메시지를 간결하고 명확하게 유지** (예: `docs: 3장 적분 제어 임계값 설명 수정`).
4. **관련 이슈와 연결** (있다면), PR 설명에 `Closes #issue번호` 사용.
5. 새로운 콘텐츠를 추가하는 경우, **기존 문서의 톤과 구조를 준수**:
   - 간결, 직접, 공학화 성향
   - 이론이 있으면 반드시 실습이 있고, 설계가 있으면 반드시 명령이 있음
6. **독립적 읽기 가능 보장**: 각 문서 파일은 독립적이고 컨텍스트 대화에 의존하지 않아야 함.

## 콘텐츠 원칙

이 튜토리얼은 다음 원칙을 준수하며, 기여 시 반드시 준수해야 함:

- **실행 가능성이 이론 완전성보다 우선**: 제공되는 콘텐츠는 독자가 즉시 조작할 수 있는 것이어야 함.
- **아키텍처가 양보기보다 우선**: 우리는 구조를 추가하는 것이 프롬프트 항목이 아니라는 것.
- **개방형 끝 허용**: 모든 문제에 최종 답이 있기를 강요하지 않지만, 조사 방향을 제시해야 함.
