# Пример таблицы интегрального контроля

Этот файл представляет фактическое состояние таблицы интегрального контроля после месяца выполнения, а также типичный диалог вывода (выводимая система).

> **Примечание** : Приведенные ниже записи являются фиктивными и служат только для иллюстрации структуры данных и логики выполнения.

## Пример запуска вывода

### Сценарий : Пользователь отмечает во второй раз "итоги AI в конце"

**Пользователь** :
> В этот раз в конце еще есть фраза "что бы ни случилось, жизнь должна продолжаться", мне не нравится подобный итог.

**Внутренний процесс Агента** :
1. Классификация ошибки → Cat A (нарушение принципа эстетики L2)
2. Поиск в таблице интеграла → находит "итоги AI в конце", количество накоплений увеличивается с 1 до 2
3. Количество = порог 2, запускает вывод

**Вывод Агента** :
> Зафиксировано. "итоги AI в конце" появилось 2 раза. Я предлагаю вывести это правило в **[L2·Подлежащая пользовательская логика] Эстетическая ориентация**, добавив запись : **"отказаться от любой формы заключения типа итог"**.
> 
> Вы согласны?

**Пользователь** :
> Согласен.

**Выполнение Агента** :
1. Запись этой записи в L2
2. Обновление таблицы интеграла : выведено → Да
3. Подтверждение : "Вывод завершен. В будущих созданиях при запуске обратной связи я буду активно загружать этот принцип."

## Пример отчёта таблицы интеграла в каждой ревизии петли фазы

> Эта таблица интеграла для этой фазы содержит 7 записей :

## Как проверить, работает ли ваша таблица интеграла правильно

Наблюдайте следующие сигналы :
- После каждого негативного возврата, классифицирует ли и накапливает ли Агент четко?
- Для записей накопленных 2 раза, предлагает ли Агент активный вывод?
- После вывода, не появляются ли больше те же типы проблем?
- В ежемесячных ревизиях петли, выведенные записи таблицы интеграла должны быть четко отслеживаемы
