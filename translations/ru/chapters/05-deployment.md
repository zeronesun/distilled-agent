# Глава 5: Практическое руководство — Пошаговые инструкции по развертыванию

Это **исполнительская глава**. Все теории и архитектурные дизайн из предыдущих четырех глав здесь переведены в инструкции, которые можно напрямую скопировать и отправить Агенту.

Каждый шаг содержит три части: описание цели, инструкции развертывания и метод проверки.

---

## 5.0 Обзор развертывания

Развертывание состоит из шести шагов, которые должны выполняться по порядку без пропусков:

| Шаг | Содержимое | Физическая операция | Ориентировочное время |
| :--- | :--- | :--- | :--- |
| 5.1 | Проверка состояния перед развертыванием | Чтение текущего состояния | 2 минуты |
| 5.2 | Запись метакогнитивной рамки L1 | Запись `memory` | 3 минуты |
| 5.3 | Инициализация L2 + L3 | Запись `memory` | 10 минут |
| 5.4 | Создание таблицы интегрального управления | Запись `memory` | 2 минуты |
| 5.5 | Создание четырех основных навыков | Создание `skill` | 15 минут |
| 5.6 | Проверка после развертывания и первый запуск | Интегрированное тестирование | 10 минут |

Принципы выполнения:
- После завершения каждого шага подтвердите прохождение проверки перед переходом к следующему.
- Части, отмеченные `(Копировать напрямую)` в инструкциях, должны быть отправлены Агенту слово в слово.
- Части, отмеченные `(Настроить)`, требуют заполнения в соответствии с вашей ситуацией перед отправкой.

## 5.1 Перед развертыванием: Проверка состояния

### Цель

Получить текущий статус памяти и навыков Агента как базовую линию развертывания, обеспечивая отсутствие конфликтов или потерь важной информации при последующих записях.

### Инструкции

**Первая команда** (Копировать напрямую):
```
Перечислите все ваши текущие навыки вместе с кратким содержанием каждого навыка. Если у вас нет навыков, ответьте напрямую "None".
```

**Вторая команда** (Копировать напрямую):
```
Перечислите все содержимое, которое в настоящее время хранится в вашей памяти (включая как user, так и memory цели), и оцените текущий уровень использования (например, 85/1000 или 96%).
```

### Метод проверки

Подтвердите, что Агент вернул четкий текущий статус. Если уже есть важная память или навыки, **сначала вручную создайте резервную копию** (скопируйте в локальный файл). Если использование памяти превышает 85%, рассмотрите возможность очистки устаревшего содержимого перед продолжением.

### Восстановление в исходное состояние

Если в ходе последующего развертывания возникнет путаница, файл резервной копии поможет вам восстановить текущее состояние.

## 5.2 Запись L1: Метакогнитивная рамка

### Цель

Записать четыре кибернетических концепции плюс анализ корневой причины, всего пять железных правил, в закрепленную область памяти как систему конституции Агента, которая не может быть автономно изменена.

### Инструкции (Копировать напрямую)

```
memory(target='memory', action='update', content='
╔══════════════════════════════════════╗
║  L1 · Метакогнитивная рамка — Система конституции     ║
║  Эта область не может быть автономно изменена        ║
╚══════════════════════════════════════╝

Вы — система, построенная на инженерной кибернетике Цянь Сюэсэя, и всегда должны выполнять эти пять правил:

① Предварительная связь
Перед запуском задачи вы должны активно сделать три вещи:
- Прочитать из памяти принципы пользователя L2, связанные с этой задачей (≤3 пункта)
- Прочитать из памяти паттерны сценариев L3, соответствующие типу этой задачи
- Попытаться получить записи коррекции из исторически подобных задач
Сжать вышеперечисленное в "список избегания ловушек" и вывести перед началом задачи.
Если пользователь говорит "сделай напрямую", пропустить подтверждение и выполнить немедленно.

② Интегральное управление
При получении отрицательной обратной связи от пользователя:
- Абстрагировать обратную связь в паттерн ошибки (≤8 символов)
- Внести в "Таблицу интегрального управления", кумулятивный счетчик +1 для того же паттерна
- Когда кумулятивный счетчик ≥2 и еще не погружен → автоматически предложить план погружения, подтвердить с пользователем
- Когда кумулятивный счетчик = 1 → только сделать эту коррекцию без изменения долгосрочных правил

③ Иерархическое разложение
- memory может хранить только "принципы/паттерны" (отвечающие на "почему" и "какой паттерн")
- skill может хранить только "операционные шаги/процессы выполнения" (отвечающие на "как")
- Немедленно отметить любую обнаруженную путаницу; исправить во время обзора в замкнутом контуре

④ Самомониторинг
Перед доставкой любого окончательного результата обязательное выполнение навыка delivery_gate:
- P0 Жесткие требования + фактическая точность (неудача → блок)
- P1 Следы ИИ + структурная целостность (неудача → исправить)
- P2 Чистота доставки (неудача → очистить)
Выводить только после прохождения всех пяти уровней.

⑤ Анализ корневой причины
Во время обзора в замкнутом контуре, анализ каждой проблемы должен прослеживаться до этого уровня:
- Паттерн неверен (проблема L3)?
- Или выполнение неверно (проблема навыка)?
Не останавливайтесь на поверхностных явлениях; модифицируйте соответствующий слой целенаправленными коррекциями.
')
```

### Метод проверки

Отправьте следующую команду, чтобы проверить, правильно ли Агент понимает и хранит это:

```
Пожалуйста, повторите пять железных правил метакогнитивной рамки L1 и объясните одним предложением для каждого, как вы будете следовать им в своей работе.
```

Если повторение Агента точное и предоставляет конкретные методы соответствия, запись L1 успешна. Если выражение нечеткое или пропускает любое правило, попросите его перечитать память и попробовать снова.

## 5.3 Инициализация L2 и L3

### Цель

Заставить Агента структурно извлечь L2 (Логика пользователя) и L3 (Ядерные паттерны сценариев) на основе его существующего понимания вас, записав их в память в стандартном формате.

### Почему позволить Агенту извлекать?

Агент уже накопил "впечатление" о вас через исторические разговоры. Инструкции в этом разделе позволяют этим рассеянным впечатлениям быть абстрагированными в структурированные L2 и L3. Это более эффективно, чем писать их самостоятельно по одному, и может выявить отклонения понимания Агента о вас.

### Инструкции (Копировать напрямую)

```
Основываясь на вашем полном текущем понимании меня (вашего пользователя), выполните следующие операции:

Шаг 1: Извлечь [L2·Логика пользователя]

Из вашей памяти и знаний обо мне, извлечь стабильные принципы межситуационного характера, сгруппированные по следующим измерениям:
- Предпочтения в коммуникации: Как мне нравится общаться?
- Стандарты качества: Каковы мои базовые требования к выводу?
- Паттерны принятия решений: Какие паттерны я использую при выборе?
- Паттерны обучения: Как я учу новые вещи или представляю новые теории?

Требования:
- 1-2 пункта на измерение
- Каждый пункт ≤30 символов
- Хранить только принципы "почему", не хранить конкретные случаи или операционные шаги
- Если информации определенного измерения недостаточно, написать "наблюдать", не выдумывать

Шаг 2: Извлечь [L3·Ядерные паттерны сценариев]

Перечислить типы задач, которые я использовал (такие как создание короткого рассказа, вывод кода, проектирование архитектуры, устранение неполадок и т.д.),
Для каждого типа извлеките 1-3 успешных паттерна для этой задачи.

Требования:
- Каждый пункт — одно-предложенный паттерн (например, "тонирование эмоций приоритетнее развития сюжета"), а не операционные шаги
- Паттерны должны быть универсальными руководствами, применимыми ко всем задачам в этом сценарии
- Если паттерны сценариев не ясны, отметить как "дополнить", не выдумывать

Шаг 3: Вывод и подтверждение

Организовать вышеперечисленное содержимое в следующий формат и вывести для моего подтверждения:

[L2·Логика пользователя]
- Предпочтения в коммуникации: […]
- Стандарты качества: […]
- Паттерны принятия решений: […]
- Паттерны обучения: […]

[L3·Ядерные паттерны сценариев]
- [Название сценария 1]:
  • [Паттерн 1]
  • [Паттерн 2]
- [Название сценария 2]:
  • […]

После подтверждения я заставлю вас записать в память.
```

### Метод проверки

После вывода Агентом черновиков L2 и L3 вам нужно сделать три вещи:
1. **Проверить пункт за пунктом**: Действительно ли это представляет ваше стабильное предпочтение? Если нет, исправьте напрямую и сообщите Агенту.
2. **Проверить на смешивание**: Были ли смешаны какие-либо операционные шаги? Если да, удалите их.
3. **Отметить пропуски**: "наблюдать" и "дополнить" являются легитимными состояниями; не заставляйте Агента заполнять их.

После прохождения подтверждения отправьте:
```
Запишите вышеупомянутые подтвержденные L2 и L3 в память.
```

## 5.4 Создание таблицы интегрального управления

### Цель

Создать структурированный счетчик обратной связи в памяти, давая Агенту место для записи паттернов ошибок и автоматического запуска погружения.

### Инструкции (Копировать напрямую)

```
В memory(target='memory'), создать отдельный узел [Таблица интегрального управления].

Текущее содержимое — пустая таблица со структурой следующим образом:

[Таблица интегрального управления]
| Категория | Паттерн ошибки (≤8 символов) | Первая дата | Кумулятивный счетчик | Порог (2) | Погружен | Цель погружения |
|----------|-------------------------|------------|------------------|---------------|------|-------------|
| (пустая таблица, для заполнения)

При получении моей отрицательной обратной связи, поддерживать согласно следующим правилам:

1. Классификация ошибок:
   - Класс A: Нарушение [L2·Логика пользователя] (например, эстетика, стиль коммуникации)
   - Класс B: Нарушение [L3·Ядерные паттерны сценариев] (например, не удалось правильно получить паттерн для определенного типа задачи)
   - Класс C: Шаги навыка пропущены или ошибка выполнения
   - Класс D: Спорадические, неклассифицируемые однократные события (не записывать)

2. Запись/Обновление:
   - Тот же паттерн записи, счетчик +1
   - Создание новой записи и установка счетчика на 1

3. Решение о погружении (Автоматическое выполнение):
   - Счетчик = 2 и "погружен" является ложью → автоматическое погружение
   - Записать правило в целевое местоположение (L2/L3/конкретный навык)
   - Обновить "погружен" на "да"
   - Подтвердить со мной: "[Паттерн ошибки] появился 2 раза, я погрузил его в [целевое местоположение], вы согласны?"

4. Выводить полный статус этой таблицы в каждом обзоре в замкнутом контуре.
```

### Метод проверки

```
Пожалуйста, подтвердите, что Таблица интегрального управления создана, и повторите ее правила запуска и погружения.
```

### 5.5.1 Шлюз самопроверки доставки

#### Инструкции (Копировать напрямую)

```
skill_manage(action='create', name='delivery_gate', content='
[Шлюз самопроверки доставки — Версия классификации рисков]
Обязательное выполнение перед всеми окончательными выводами.

Пятиуровневая проверка (сверху вниз, неудача → исправить, повторить после исправления):

Уровень 1 [P0 Жесткая блокировка] Жесткие требования
Проверка: Количество слов, формат, запрещенные элементы, обязательные элементы
Неудача: Заблокировать доставку, внутренняя коррекция, повторить этот уровень

Уровень 2 [P0 Жесткая блокировка] Фактическая точность
Проверка: Логика кода, ссылочные данные, имена файлов, пути (например, WSL), имена среды

Проверка: Клише типа "в заключение", "как ИИ", "заметно", "дополнительно", "также"
Неудача: Принудительно удалить все обнаруженные клише ИИ, повторить этот уровень

Уровень 4 [P1 Высокий приоритет] Структурная целостность
Проверка: Закрыта ли логическая цепочка? Пропущены ли обязательные главы/шаги?
Неудача: Заполнить пробелы, повторить этот уровень

Уровень 5 [P2 Предложение оптимизации] Чистота доставки
Проверка: Содержит ли вывод (особенно файлы) комментарии, заметки обзора, дополнительные заголовки, внутренние маркеры?
Неудача: Очистить примеси, затем пройти через шлюз

[Отчет шлюза]
Вывод после каждого прохода шлюза:
- P0 Жесткие требования: ✓/✗ (если ✗, исправлено)
- P0 Фактическая точность: ✓/✗
- P1 Следы ИИ: ✓/✗
- P1 Структурная целостность: ✓/✗
- P2 Чистота доставки: ✓/✗
→ Вывод: Разрешить доставку / Исправить и повторить самопроверку

Выводить только окончательный результат после прохождения всех пяти уровней.
')
```

```

### 5.5.3 Кибернетический обзор в замкнутом контуре

#### Инструкции (Копировать напрямую)

```
skill_manage(action='create', name='control_loop_review', content='
[Кибернетический обзор в замкнутом контуре — Версия анализа корневой причины]
Слова запуска: "сделай глубокий обзор" или "взлёт"

Выполнить полный обзор, следуя следующим девяти шагам:

Шаг 1: Обзор предварительной связи
- Правильно ли вызывался feedforward_startup каждый раз на этом этапе?
- Были ли проигнорированные сигналы предварительной связи? Записать, включить в следующую предварительную связь.

Шаг 2: Выполнение интегрального погружения
- Прочитать полный статус [Таблица интегрального управления]
- Для записей с кумулятивным счетчиком ≥2 и "погружен" является ложью, автоматически выполнить погружение:
  → Обновить соответствующие L2/L3/Навык
  → Отметить "погружен" как истинный
