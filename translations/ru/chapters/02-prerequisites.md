

# Глава 2: Применимые условия и подготовка

## 2.1 Минимальные требования платформы агента к этому методу

Прежде чем приступить к работе, убедитесь, что ваша платформа агента обладает следующими возможностями. Это физический фундамент для реализации архитектуры.

### 2.1.1 Обязательные возможности (без одной — невозможно полностью реализовать)

| Возможность               | Описание                                   | Использование                                                     |
| :----------------- | :------------------------------------- | :------------------------------------------------------- |
| **Персистентная память**     | Система памяти для хранения, чтения, обновления между сессиями       | Носит L1 метакогнитивный фреймворк, L2 логика пользователя, L3 законы сцен, таблица интегрального управления |
| **Процедурные модули**     | Создаваемые, вызываемые, обновляемые навыки/Skills/GPTs и т.д. | Носит уровень исполнения (шлюзы доставки, прямой запуск, циклический обзор и т.д.)             |
| **Достаточное контекстное окно** | Рекомендуется 100k токенов и более                  | Вмещает список прямого управления, записи избегания ям, процесс выполнения задач в сессии             |

Если ваша платформа имеет все три вышеуказанные возможности, можно полностью реализовать все содержание этого руководства.

### 2.1.2 Бонусные возможности (лучше есть, нет — работает пониженного уровня)

| Возможность         | Описание                                         | Альтернатива при отсутствии                                 |
| :----------- | :------------------------------------------- | :----------------------------------------------- |
| **Поиск по сессиям** | Искать содержимое и записи исправлений в исторических сессиях               | Вручную записывать ключевые уроки в память после каждого исправления, как материал прямого управления |
| **Вызов инструментов** | Агент может активно вызывать внешние инструменты (поиск, выполнение кода и т.д.) | Ограничены по масштабу, но основная архитектура не затрагивается                       |

### 2.1.3 Что делать если платформа не имеет функции Skill

Некоторые платформы (например, базовая версия ChatGPT, простые чат-интерфейсы) не имеют модуля навыков. В этом случае можно использовать **«метод симуляции памяти»**:

- Содержимое Skill хранить как структурированные записи памяти, например `[Skill-шлюз доставки] Содержание: ...`
- В L1 метакогнитивный фреймворк добавить обязательное правило: «Перед каждой доставкой, необходимо читать и выполнять все записи памяти, помеченные [Skill-*]»
- Недостатки: занимает емкость памяти, и агент должен активно искать; преимущества: логика полностью сохранена, подходит для сценариев с легким использованием.

## 2.2 Самооценка: подходит ли вам это руководство

### 2.2.1 Портрет пользователей, которые крайне рекомендуются

- **Пользователи высокоточного сотрудничества**: у вас есть стабильные требования к качеству вывода агента (например, стиль письма, спецификации кода)
- **Пользователи повторных задач**: вы часто сотрудничаете с агентом в схожих задачах (например, ежедневный/еженедельный рабочий процесс)
- **Пользователи с системным мышлением**: вы готовы инвестировать в начальное построение, взамен долгосрочной эффективности сотрудничества
- **Пользователи с четкими границами**: вы можете четко экспрессировать предпочтения и красные линии, когда агент совершает ошибку, вы готовы указать

### 2.2.2 Не слишком подходящие пользователи

- Только операции одноразовых, без потребности в повторном использовании случайных вопросов
- Сценарии использования покрывают очень широкий круг, трудно суммировать стабильные законы
- Не желают устанавливать долгосрочные отношения сотрудничества с агентом

## 2.3 Подготовка до начала

Прежде чем официально развертывать, завершите следующие четыре подготовки, это может значительно снизить последующие издержки корректировки.

### 2.3.1 Инвентаризация текущего состояния агента

Выполните следующие две команды, запишите текущее состояние как базовый уровень:

```
Выведи все ваши текущие skill, и их краткое содержание.
```

```
Выведи все содержимое, хранящееся в вашей памяти, и оцени использование.
```

**Ключевые точки записи**:
- Сколько сейчас skill? Все ли они еще используются?
- Использование памяти сколько процентов? Содержание ли уже смешалось?
- Есть ли много устаревшей информации?

### 2.3.2 Сформулировать предпочтения и правила, которые вы хотите закрепить

Используйте простую таблицу, перечислите принципы, которые вы хотите, чтобы агент соблюдал в долгосрочной перспективе. Этот шаг помогает вам из неопределенного «ощущения» выделить операционованные правила:

| Размер     | Ваши принципы               | Пример                   |
| :------- | :--------------------- | :--------------------- |
| Стиль коммуникации | Сжатый и прямой, не нужно вежливых вежливых фраз   | Не выводить «надеюсь помочь вам» |
| Формат вывода | Код приоритет, с методами проверки | При выводе схемы включать шаги тестирования |
| Предпочтение решений | Риски вперед, сначала огласить проблемы   | Сначала перечислить риски потом говорить о выгоде       |
| Эстетическая ориентация | Холодное повествование, против ИИ-подытогов | Конец рассказа оставить пробел, не возвышение   |

Не нужно исчерпать все, 4-6 ключевых принципов достаточно. Впоследствии агент будет постоянно добавлять в цикле.

### 2.3.3 Вспомнить 3-5 типов задач, которые вы часто выполняете

- Например: писать короткие рассказы / обзор кода / операционная проверка / архитектурный дизайн / анализ данных
- Для каждого типа попробуйте написать одно предложение, описав ваш ключевой закон «как сделать правильно» для этого типа задач

**Примеры**:
- Создание коротких рассказов: «Эмоциональная настройка имеет приоритет над сюжетным развертыванием, конец должен оставлять пробел.»
- Обзор кода: «Сначала найти архитектурные проблемы, потом смотреть номенклатурные стандарты; для каждого типа проблемы дать рекомендации по изменению.»
- Проверка проблем: «Сначала смотреть логи для определения коренной причины, не гадать по опыту; когда не уверены, отметить 'ожидает проверки'.»

Эти станут начальным содержанием ваших L3 законы сцен.

### 2.3.4 Резервное копирование текущего состояния

Перед началом развертывания, **сделайте резервную копию текущих состояния памяти и навыков**. Метод зависит от вашей платформы:

- Если платформа поддерживает экспорт, сначала экспортируйте копию
- Если не поддерживает, вручную скопируйте текущие списки памяти и навыков в локальный документ
- Если есть управление версиями (например, некоторые платформы поддерживают snapshots памяти), создайте точку восстановления

Цель этого шага: на случай если в процессе развертывания возникнет путаница, вы можете восстановиться в исходное состояние и начать заново.

## 2.4 Инойндикатор завершения подготовки

Когда вы можете ответить на следующие три вопроса, фаза подготовки завершена, можно переходить в третью главу:

1. Обладает ли моя платформа агента персистентной памятью + модуль навыков? (Да / Нет / Частично)
2. Я обдумал несколько ключевых предпочтений и принципов? (Рекомендуется ≥4)
3. Я перечислил несколько типов высокочастотных задач, и для каждого типа есть предварительное «суждение» о законах? (Рекомендуется ≥3)
