# Руководство по вкладу

Спасибо за ваш интерес к проекту **Cybernetic Your Agent**. Цель этого проекта — сделать идеи инженерной кибернетики Цянь Сюэсэня доступными для легкого развертывания и использования каждым, кто работает с современными AI-агентами.

Мы приветствуем вклад в любой форме.

---

## направления вклада

Вот как вы можете внести свой вклад в этот проект:

### Дополнительные практические примеры

Разверните эту архитектуру на различных платформах агентов и поделитесь своим опытом:
- Особенности адаптации для конкретной платформы (синтаксис вызова инструментов, различия в механизмах памяти и т.д.)
- Проблемы, возникшие на практике, и их решения
- Обратная связь о результатах после запуска в течение некоторого времени (коэффициент повторных исправлений, вероятность попадания прямых связей и т.д.)

### Теоретическое расширение

Внесите новые теоретические рамки для расширения L1 метакогнитивного слоя:
- Например: байесовский вывод, информационная теория Шеннона, ТРИЗ, теория ограничений и т.д.
- Требование: объясните, как новая теория работает в сочетании с существующими четырьмя концепциями, и предоставьте исполняемые пункты расширения L1

### Переводы

Переведите учебник на другие языки. Пожалуйста, сначала создайте Issue, указав язык, который вы собираетесь перевести, чтобы избежать дублирования работы.

### Улучшение документации

- Исправление опечаток, грамматических ошибок, неясных формулировок
- Добавление диаграмм (архитектурные схемы, диаграммы потоков данных)
- Улучшение примеров файлов

### Код и скрипты

- Предоставьте автоматические скрипты развертывания для конкретных платформ агентов
- Предоставьте инструменты для экспорта состояния памяти/навыков

## Предоставление стандартов

### Issue

- Перед отправкой отчета об ошибке, предложения по улучшению или новой идеи, пожалуйста, сначала найдите существующие Issue.
- Четко опишите проблему или предложение, приложите соответствующий контекст.

### Pull Request (Пулл-реквест)

1. **Сделайте форк этого репозитория**.
2. **Создайте отдельную ветку** для новой функции или исправления (например, `feat/add-claude-mcp-guide`).
3. **Держите информацию о коммите краткой и ясной** (например, `docs: исправьте описание порога интегрального управления в главе 3`).
4. **Свяжите с соответствующим Issue** (если таковой имеется), используя `Closes #issue` в описании пулл-реквеста.
5. При добавлении нового контента **следуйте тону и структуре существующей документации**:
   - Консективно, прямо, с упором на инженерный подход
   - Если есть теория, должна быть практическая реализация; если есть дизайн, должны быть инструкции
6. **Обеспечьте возможность независимого чтения**: каждый файл документации должен быть самодостаточным и не зависеть от контекста диалога.

## Принципы контента

Этот учебник придерживается следующих принципов; пожалуйста, соблюдайте их при внесении вклада:

- **Исполняемость приоритетнее теоретической завершенности**: предоставляемый контент должен быть немедленно применим читателем.
- **Архитектура приоритетнее объема**: мы добавляем структуру, а не пункты промптов.
- **Открытые конечные значения допустимы**: не требуется окончательного решения для каждой проблемы, но должны быть указаны направления для устранения неполадок.

## Кодекс поведения

- Обсуждения сосредоточены на самом учебнике, оставаясь уважительными и конструктивными.
- Приветствуются рациональные обсуждения различных мнений, личные нападки не приветствуются.
- Сопровождающие оставляют за собой право закрывать Issue/пулл-реквесты, которые отклоняются от темы или не соответствуют стандартам.
