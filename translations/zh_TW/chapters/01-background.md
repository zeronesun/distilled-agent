# 第一章：背景與核心理念

### 1.1 現狀：AI Agent 協作普遍面臨的三大瓶頸

現代 AI Agent 已經擁有了持久化記憶和可調用技能，但在長期、高精度協作中，它們往往會陷入以下困境：

- **重複糾錯循環**：Agent 在每個新 session 中反覆犯同類錯誤，使用者需要反覆糾正，形成「錯誤-糾正-遺忘-再錯誤」的循環。
- **記憶侵蝕決策品質**：記憶逐漸膨脹成「大雜燴」，事實、偏好、操作步驟混雜在一起，導致訊號雜訊比急劇下降，Agent 的行為變得不穩定、容易偏差。
- **回饋過激或回饋遲鈍**：Agent 要麼對一次偶然吐槽就大幅度改變行為（震盪），要麼對重複出現的真實偏好無動於衷，缺乏穩定的敏感度閾值。

這些瓶頸的本質原因在於：**Agent 缺少一個穩定、可自糾錯的系統架構，其記憶和技能沒有按照「控制論」的原則進行組織和進化。**

### 1.2 解決思路：不教新知識，而是搭建架構

本教程提供的不是一套新 prompt 或某個神秘技巧，而是一種 **系統級的記憶與技能組織方式**。通俗地說，就是把錢學森《工程控制論》中最具有工程落地價值的幾個概念，適配到現代 AI Agent 的架構上，為 Agent 建立一個能自我進化、抗雜訊、可長期穩定的「元認知框架」。

其核心思想是：**你不是在訓練一個更聰明的 AI，而是在設計一個能與你對齊原則、並持續優化的協作系統。**

### 1.3 根基：錢學森工程控制論的四個可執行概念

我們提取並簡化為 Agent 可執行的四個核心概念，它們將構成 Agent 的底層作業系統（L1 元認知框架）：

| 概念                                       | 作用                     | 一句話落地                                                   |
| :----------------------------------------- | :----------------------- | :----------------------------------------------------------- |
| **前饋（Feedforward）**                    | 把錯誤堵在發生之前       | 每次任務啟動時，主動載入使用者偏好、場景規律、上次同類錯誤記錄，不等犯錯先避坑。 |
| **積分控制（Integral Control）**           | 實現穩定敏感的回饋閾值   | 同類負面回饋出現 ≥2 次，才把規則沉降到長期記憶或技能裡；單次回饋只做臨時修正。 |
| **層級分離（Hierarchical Decomposition）** | 防止記憶混亂、提升訊號雜訊比 | 記憶體只存「為什麼」（原則、規律），技能只存「怎麼做」（操作步驟、檢查清單）。 |
| **自監控（Self-Monitoring）**              | 交付前自動質檢，自動糾偏 | 交付任何結果前，內嵌一個閘門自動掃描硬性要求、AI 痕跡、結構完整性等，不全過不輸出。 |

> **註**：此處「積分控制」中的「積分」一詞，借用了控制理論中積分器的概念——對偏差進行累積，超過閾值才觸發動作。它恰好回答了「如何讓 Agent 既能過濾偶發雜訊、又不錯過真實偏好」這個工程問題。

### 1.4 三層邏輯及運作原理速覽

在物理實作上，我們利用 Agent 平台已有的 `memory`（記憶）和 `skill`（技能）功能，將它們重組為三個邏輯層次：

- **L1 元認知框架**（存於記憶的最優先級）
  上面四個控制論概念的具體行為準則。它作為系統憲法，不可被 Agent 自行修改，是所有記憶和技能運作的「校準器」。

- **L2 使用者底層邏輯 + L3 場景核心規律**（存於記憶的主體區）
  它們共同回答「為什麼這麼做」和「這類任務應該遵循什麼規律」。例如：
  * L2：「使用者偏好冷感敘事，排斥 AI 式總結。」
  * L3：「短篇創作的核心規律——情緒定調優先於情節，結尾留白。」

- **執行層 Skill**（存於技能模組）
  由 L3 規律驅動，並內嵌 L1 控制行為的操作流水線。例如一個「短篇創作 Skill」，它會在執行前主動載入 L2 審美、執行後自動過自監控閘門。

整個系統按照以下循環運作：

1. **啟動任務** → 前饋：載入 L2、L3、歷史糾正記錄，生成避坑清單。
2. **執行任務** → 層級分離：記憶只出方向，Skill 只出執行。
3. **交付結果** → 自監控：經過交付閘門的多關檢查，不過即修。
4. **使用者負面回饋** → 積分控制：同類累加至閾值，自動下沉為新的規則或修改技能。
5. **週期性複盤** → 用 L1 的四條標準審計整個系統，更新 L2/L3/Skill，完成閉環進化。

### 1.5 適用場景與前提條件

本教程最適合下列協作場景：
- 長期、高精度的任務（如深度寫作、持續化軟體開發、專案管理）；
- 需要 Agent 保持穩定輸出風格和品質標準的場合；
- 使用者有明確的偏好或工作流程，並希望 Agent 自動適應和維護這些規則。

使用本教程，你的 Agent 平台需要具備：
- 可編輯的持久化記憶（跨 session 儲存）；
- 可建立/呼叫的技能或程序性模組；
- 足夠的上下文視窗（建議 100k tokens 以上，視任務複雜度而定）；
- （可選）session 檢索能力，能顯著增強前饋效果。

如果你的平台不具備內建 Skill，也可以使用「記憶條目 + 固定系統指令」的方式模擬，效果略降階，但仍能搭建核心的積分控制和層級分離邏輯。

### 1.6 你將獲得什麼

完成訓練後，你的 Agent 將從「不斷需要糾錯的工具」轉變為一個：

- **會主動避坑**（啟動前給你檢查清單和風險預判）；
- **不亂改規則**（你吐槽一次不改，兩次才系統性地改）；
- **記憶乾淨**（只存原則和規律，不存過期細節）；
- **交付不摻水分**（自動查錯、自動淨化輸出）；
- **能自我進化**（每次複盤自動優化自己的 Skill 和記憶）。
