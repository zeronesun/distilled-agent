# L2/L3 初始化輸出範例

本檔案展示一次典型的 L2/L3 初始化對話。使用者在發送提煉指令後，Agent 基於對使用者的歷史了解，輸出結構化的 L2 和 L3 草案。

> **說明**：本範例中的使用者偏好為虛構，僅用於展示輸出格式和提煉粒度。你的 Agent 會根據你們的歷史對話生成屬於你的版本。

---

## Agent 的提煉輸出

基於我對妳的當前了解，以下是提煉結果：

### [L2·使用者底層邏輯]

- **溝通偏好**：簡潔直接，不要客套鋪墊；邊界清晰，說「不」即停
- **品質標準**：程式碼優先附帶驗證方法；輸出須可復用，不滿足一次性正確
- **決策模式**：風險前置，先暴露完整問題再談方案；二階評估優先於直覺
- **學習模式**：跨領域理論遷移；知行合一，理解必須落地為操作

### [L3·場景核心規律]

- **短篇創作**：
  - 情緒定調優先於情節鋪展
  - 結尾留白，不做價值總結或道德昇華
  - 白描建立畫面感，不依賴比喻渲染

- **程式碼輸出**：
  - 先寫測試或驗證方法，再寫實作
  - 功能完整優先於註釋詳盡
  - 不確定的行為標註「待驗證」，不硬編

- **架構設計**：
  - 先定義系統邊界與資料流，再選技術堆疊
  - 評估擴展性和失敗模式，不只寫 happy path
  - 輸出包含風險分級（P0-P3）及應對

- **問題排查**：
  - 先定位日誌/運行現場根因，不憑經驗猜測
  - 不確定處標註「待驗證」
  - 給出可復用的排查步驟，不只給結論

---

## 使用者可以做審核動作

1. **逐條確認**：每條原則問自己——「這真的是我的穩定偏好嗎？」
2. **修正偏差**：直接告訴 Agent 哪條不對，應該改成什麼
3. **補充缺失**：如果你覺得某個維度/場景遺漏了關鍵原則，直接說
4. **刪除不實**：如果是 Agent 硬編的（你沒有這個偏好），直接刪掉

---

## 常見修正範例

| Agent 的原始提煉   | 使用者的修正                               | 說明                               |
| :----------------- | :--------------------------------------- | :--------------------------------- |
| 溝通偏好：簡潔直接 | 溝通偏好：簡潔但必須有充分上下文         | 使用者發現自己需要上下文，不只是短   |
| 短篇：結尾留白     | 短篇：結尾留白；但系列故事可以有溫和收束 | 使用者補充了例外情況                 |
| 程式碼輸出：測試優先 | 程式碼輸出：快速原型時可以後補測試         | 使用者區分了原型和正式輸出的不同標準 |

## 寫入指令

審核通過後，發送：
> 將以上確認後的 L2 和 L3 寫入 memory。

Agent 會將它們存入記憶的主體區，按 [L2·使用者底層邏輯] 和 [L3·場景核心規律] 分節點存儲。
