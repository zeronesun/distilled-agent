# 序章：為什麼 AI 學會了所有知識，卻依然學不會「懂你」

你是否有過這樣的經歷：

你和一個 AI 助手已經合作了幾十次。你反覆告訴它「結尾不要總結昇華」「程式碼先寫測試再寫實作」「方案先列風險再談收益」。每次說，它都應承。每次新開一個會話，一切歸零。下次你發同樣的任務，它熱情地又犯了一遍同樣的錯，等著你再一次糾正。

**問題不在於 AI 不夠聰明。** 它的模型參數可能有數千億。它讀過人類有史以來幾乎所有的公開文本。你告訴它的每一句話，它都「理解」。但它就是記不住——不是記不住資訊本身，而是記不住**什麼對你重要、什麼在你這裡算是犯錯、什麼樣的交付算合格**。

市面上解決這個問題的主流思路是：繼續教。寫更長的 prompt。整理更詳細的規則文件。搞一套複雜的 RAG 知識庫。每次 AI 犯錯就再補一條規則進去，直到系統提示詞膨脹到幾萬 token，規則之間開始互相矛盾，而妳本人也記不清自己到底寫過多少條要求。

**這本教程走一條相反的路。**

我們不教 AI 新知識。我們**用 20 世紀最硬核的系統工程理論，為 AI 重新設計記憶與技能的架構**。

這個理論的名字聽起來很遙遠——錢學森《工程控制論》。但妳不需要是工程師，也不需要懂數學。妳只需要理解四個概念，並按照本教程的步驟在妳的 AI Agent 上部署：**前饋**（不等錯誤發生先避坑）、**積分控制**（吐槽一次不改，兩次才系統性改）、**層級分離**（原則和步驟徹底分家）、**自監控**（交付前自動品質檢驗）。

這四個概念，加上一層週期性閉環反省，就能讓妳的 Agent 從「每次都要重新教的工具」，變成一個**主動避坑、不亂震荡、持續對齊妳原則的協作引擎**。

這個思路不是坐在書齋裡想出來的。

它始於一條筆記。在那條筆記裡，一個使用者嘗試把錢學森工程控制論的核心思想「投餵」給他的 Hermes Agent，用來組織記憶體和技能層，讓 Agent 工作時按這套邏輯觸發對應能力。評論區一片「怎麼搞」「試了後還真有用」。

這本教程，就是將那條筆記中的洞見，結合後續無數次的實測打磨和架構推演，**系統化為一套完整、可複製、面向通用 Agent 平台的實作方案**。我們從「蒸餘人的思考模式」這句話出發，一直走到能用控制論四概念驅動 Agent 自我進化的完整架構。

妳不需要特定的模型。妳不需要 Hermes。妳需要的只是一個具備**持久化記憶**和**技能/程序性模組**的現代 Agent 平台，以及一份願意用一兩個小時搭建一套長期回報系統的耐心。

## 這本書不是什麼

**這不是一本 prompt 合集。** 妳在這裡找不到「讓 AI 寫作更好的 50 條提示詞」。我們在乎的是架構，是系統，是怎麼讓提示詞的組織方式本身變得可控和可進化。

**這不是一篇學術論文。** 我們對工程控制論的引用是高度工程化、高度簡化的。錢學森的體系遠比這裡呈現的深廣。我們只取了四個對 Agent 架構設計有直接指導價值的概念，並做了一件事：**把它們變成 Agent 可執行的指令。**

**這不是一次性改造。** 這套系統的核心特徵是「閉環進化」——部署只是開始，不是終點。妳每週說一次「起飛」，Agent 會用控制論標準自我審計，不斷靠齊妳的真實偏好。

## 這本書適合誰

如果妳符合以下任何一條，這本書可能對妳有用：

- 妳有一個經常協作的 AI Agent，但每次新會話都像是在重新認識它。
- 妳已經厭倦了反覆糾正同一個問題，希望糾正一次就能沉淀為長期規則。
- 妳對「把系統工程思想用在 AI 上」這件事本身感到好奇。
- 妳不滿足於把 AI 當成一個問答工具，想把它變成一個有穩定個性和自我糾偏能力的協作者。

如果妳只是偶爾用 AI 聊聊天、問問一次性問題，這本書的架構對妳來說可能太重了。妳可以讀前三章了解思想，後面的實作不必全量部署。

## 如何使用這本書

本書共七章，按照「為什麼 → 是什麼 → 怎麼搭 → 怎麼用 → 怎麼修」的路徑組織：

- **第一章到第四章**是理論基礎和架構設計。如果妳讀完後覺得「這個思路我認可」，再進入實作。
- **第五章**是純指令操作章。妳可以直接複製貼上指令發給妳的 Agent，按步驟完成部署。
- **第六章**教妳日常怎麼用、怎麼驗證系統在正常運行。
- **第七章**是排障和延伸思考，碰到問題隨時回查。

如果妳使用的 Agent 平台是 Hermes（或類似的能力集合），可以幾乎無縫執行全部步驟。如果妳的平台能力有差異，第七章的 7.1 節提供了降階適配方案。

## 開始之前，先說清楚一件事

部署這套系統需要妳花一個小時。回報是：此後每一次協作，Agent 都會主動回憶妳的偏好、主動避開已知的坑、在交付前自我品質檢驗，並在每週復盤時自動優化自己的行為。

**這不是讓 AI 更「強」。是讓它更「懂你」。**

兩者之間的差距，就是一百條 prompt 和一個架構之間的差距。
