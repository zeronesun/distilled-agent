# 貢獻指南

感謝妳對 **Cybernetic Your Agent** 專案的關注。本專案的目標是讓錢學森工程控制論的思想，能夠被任何使用現代 AI Agent 的人輕鬆部署和受益。

歡迎任何形式的貢獻。

---

## 貢獻方向

以下是妳可以為這個專案添磚加瓦的方式：

### 補充實踐案例

在不同的 Agent 平台上部署本架構，並分享妳的經驗：
- 平台特有的適配方式（工具呼叫語法、記憶機制差異等）
- 實踐中遇到的問題和解決方案
- 運行一段時間後的效果反饋（重複糾正率、前饋命中率等）

### 理論延展

引入新的理論框架來擴展 L1 元認知層：
- 例如：貝葉斯推理、香農資訊論、TRIZ、約束理論等
- 要求：說明新理論如何與現有的四概念協同工作，並給出可執行的 L1 擴展條目

### 翻譯

將教程翻譯為其他語言。請先開 Issue 聲明妳要翻譯的語言，避免重複工作。

### 文檔改進

- 修正錯別字、語法錯誤、表述不清
- 補充圖表（架構圖、資料流圖）
- 改進範例檔案

### 程式碼與腳本

- 為特定 Agent 平台提供自動部署腳本
- 提供記憶/技能狀態匯出的工具

## 提交規範

### Issue

- 提交 Bug、改進建議、新想法之前，請先搜尋已有 Issue。
- 清晰描述問題或建議，附上相關上下文。

### Pull Request

1. **Fork 本倉庫**。
2. 為新功能或修復**建立獨立分支**（例如 `feat/add-claude-mcp-guide`）。
3. **保持提交資訊簡潔明確**（例如 `docs: 修正第三章積分控制閾值說明`）。
4. **關聯對應 Issue**（如果有的話），在 PR 描述中使用 `Closes #issue號`。
5. 如果是新增內容，請**遵循現有文檔的語氣和結構**：
   - 簡潔、直接、偏工程化
   - 有理論必有落地，有設計必有指令
6. **確保可以獨立閱讀**：每個文檔檔案應該是自包含的，不依賴上下文對話。

## 內容原則

本教程堅持以下原則，請貢獻時務必遵守：

- **可執行優先於理論完備**：給出的內容必須是讀者可以立刻操作的。
- **架構優先於堆量**：我們增加的是結構，不是 prompt 條目。
- **開放式結尾被允許**：不強求每個問題都有終極解，但要給出排查方向。

---

## 行為準則

- 討論聚焦於教程本身，保持尊重和建設性。
- 歡迎不同意見的理性討論，不歡迎人身攻擊。
- 維護者保留關閉偏離主題或不遵循規範的 Issue/PR 的權利。
